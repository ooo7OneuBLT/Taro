# プロジェクト「太郎」

**人間の発達段階を、生後から1段ずつ再現する**ことで、人を理解し寄り添うAIのあり方を探る個人研究プロジェクトです。

## 概要

既存の大規模言語モデル（LLM）は膨大な知識を扱える一方、目の前の相手を深く理解し、その人に合わせて関わる力にはまだ課題がある。

そこで「太郎」では、既存のモデルのように大量の正解データを教え込むのではなく、**赤ちゃんが生まれつき持つ本能と、身体・環境との相互作用だけ**を与え、人間と同じ順序で能力が**自ら創発する**かを一段ずつ検証する。

検証シミュレーションの際は親シミュレーションと太郎の会話をベースに、Forループの中で1tickごとに処理を行う。

つまり、人間の一生をゼロからシミュレーションするという試みである。

### 最終目標（長期ビジョン）

将来フィジカルAIが普及し、身近になることがほぼほぼ確定している中で工場や建設の現場では関係ないかもしれないが、介護や先生などの人間と深く関わり合う場合は、上記の課題が浮き彫りになると考えた。そのため、将来そのようなAIのソフトウェアの部分につながることを期待して、本研究をスタートさせた。

### 発想

以下の３つの要素・経験からこのプロジェクト/研究の発想を得た。

* 軽度の認知症を抱える祖母の介護⇒人に寄り添う難しさを感じた
* ゲーム『Detroit: Become Human』をプレイしたこと。(2038年のデトロイトを舞台に、人間とほぼ見分けのつかないアンドロイドを描いた物語性の強いゲーム)⇒アンドロイドと人間の摩擦が印象的で、こうした未来が実際に来るかもしれないと考えた
* フィジカルAI関連の産業動向を調べる中で、日本はハードウェアでは有望だが、大規模言語モデルのソフトウェア(例えばopenAIのChatGPTやCodex)ではかなり遅れを取っていると感じたこと⇒一方で今までとは違う(認知発達ロボティクスのような)アプローチは日本で活発であり、この研究を社会実装することが重要

### 短期目標（マイルストーン）

* **目標A：オウム返し**（親の発声を模倣する段階）… ✅ 達成
* **目標B：初語**（音に意味がつく＝理解と、自分からの発話）… 🔧 進行中（核心は達成）

いずれも「答えを直接教えず、人間が持つ仕組みだけを与えて創発させる」という方針（下記「哲学」）を一貫して守っている。各段階の詳しい達成状況は「発達段階の状況」を参照。

## 哲学

* 太郎の内部には**人間の乳児が生まれつき持つ本能だけ**を実装する
* 人間に存在しない便利機能（正解ラベルの注入、決め打ちの数式やしきい値など）は使わない
* すべての機能に根拠ラベルを付ける：**【人間模倣】** or **【既存AI研究】**（人間模倣から逸脱する場合は⚠️で明記）
* 学習の工夫は太郎の内部ではなく「環境（親の話し方・育て方）」側に置く
* 壁にぶつかったら「太郎と人間の赤ちゃんの違いは何か」から出発して解決策を探す
* 「思っていたのと違う結果」も削らずそのまま記録する（[人間模倣からの逸脱リスト](B/docs/人間模倣からの逸脱リスト.md)）

## 前提

* 太郎は**日本語が母語の日本人・男児**の設定
* 長期ビジョン：人間と同じように育てたAIは人間社会になじめるのではないか

## リプレイビューア

太郎のシミュレーションは長期間（最大2年分）を高速で回すため、内部で何が起きているかが見えにくくなります。
そこで、走行中に書き出したログ（`trace.jsonl`）を読み込み、**動画プレイヤーのように再生・巻き戻し・
場面ジャンプ**しながら内部状態を可視化するツールを自作しました。

**▼ 実際の動作の様子**

https://github.com/user-attachments/assets/4e64f01a-25a0-4e21-8b39-7648d65fc64f

画面は4つのパネルで構成されます：

* **体ビュー**：2Dの人体図。胃・肺・声道や脳の各部位が、その瞬間に**実際に活動している度合い**に応じて
光ります（決め打ちの対応表ではなく、シミュレーションの実測値から算出）。
* **ネットワーク図**：脳内の各部位（大脳皮質・小脳・青斑核・海馬・島皮質など）をノードで表し、
実際に活動した部位とそのつながりを強調します。
* **メトリクス**：空腹・眠気・不快・ノルアドレナリン（探索度）・ドーパミンなどの数値を時系列ゲージで表示します。
* **タイムライン**：0歳〜2歳を日付でスクラブでき、任意の瞬間（例：生後6か月ごろの初めての「まんま」）へ飛べます。

これにより「**いつ・どの部位が働き・情報がどう流れ・何を発話したか**」を1イベントずつ追えます。
`replayViewer/ビューアを開く.bat` から起動すると、ローカルHTTPサーバが立ち上がりブラウザで開きます。
設計の詳細は[replayViewer/docs/構想_リプレイビューア.md](replayViewer/docs/構想_リプレイビューア.md)を参照してください。

## 開発体制

本研究は高校生1人による個人研究です。実装・文献調査・分析には
[Claude Code](https://claude.com/claude-code)（Anthropic社のAIコーディングエージェント）を
ペアプログラマーとして使用しています。研究の方向性・設計判断・「何を検証するか」の**最終判断は
著者本人が行い**、Claude Codeは実装計画の精査・コード実装・文献の一次調査/翻訳・シミュレーション実行・
結果の集計・研究の自動メモを担当し、加えて方針を考える際の\*\*相談相手（壁打ち）\*\*にもなっています。
研究日誌（[A](A/docs/研究日誌.md) / [B](B/docs/研究日誌.md)）には両者のやり取りの経緯もそのまま記録しています。

## 動かし方（Getting Started）

**必要環境**：Python 3.12（動作確認は3.12。3.10以上であれば動くはずです）。

**依存ライブラリ**：

|ライブラリ|用途|
|-|-|
|**PyTorch**|太郎の脳（GRU）と学習|
|**PyYAML**|設定ファイルの読み込み|
|**matplotlib**|学習曲線などのグラフ出力|
|**numpy**|数値計算|
|psutil（任意）|CPUスレッド数・優先度の最適化。無くても動作する|

```bash
pip install -r B/requirements.txt
```

**短いシミュレーションを1本回して、リプレイで見る**（目標B）：

```bash
cd B
python run\\\_trace.py 7        # 生後7日分をシミュレーション（数十秒程度）
```

`replayViewer/data/trace\\\_7d/` にログが出力されます。`replayViewer/ビューアを開く.bat` を実行し、
ビューア下部の「ファイルを選択」でこのフォルダを選ぶと、シミュレーションを再生できます。

各スクリプト・モジュールの役割は [**コード構成.md**](コード構成.md) を参照してください。
