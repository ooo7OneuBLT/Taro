### 考察

#### モデルAは新生児期をすっ飛ばしていた

モデルAには内部状態（空腹・つらさ）が無く、泣きもなかった。いきなりクーイング（母音）から始めていた。人間の発達でいうと生後2-3か月からのスタート。モデルBはゼロ（誕生＝泣きだけ）からやり直す。

#### 感情は最初「1次元」しかない

Bridges（1932）の感情分化説を採用。生まれた瞬間は「つらさ」の1次元だけ。「うれしい」や「こわい」は後で経験から枝分かれする。だから太郎にも最初は感情の枠すら作らない。

#### シミュレーションの解像度ルールを明文化した

- カーブは数式で決め打ちしない。メカニズムを作って結果を出す
- 臓器を増やすのではなく、臓器間の接続を足す
- 脳に渡す信号が変わる部分だけ精密にする
- 最初は粗く、合わなくなったら1段階ずつ上げる

#### オウム返しはBでも出る（ただし10回連続の判定基準を満たすのが難しい）

Bの構成でゼロから学習したところ、オウム返し自体はできた（転移テスト0.90-0.99）。ただし体の状態（つらさ）の変動で集中できず、10回連続の判定が通らなかった。能力自体は獲得している。

#### 未実装：穏やかな時間の自発的な発声

今の太郎は親に話しかけられたときしか声を出さない。人間の赤ちゃんは起きている穏やかな時間に自分で「あーあー」「ばばば」と声を出して口の使い方を練習している。これが足りないため、1日あたりの発声練習回数が少なすぎる（34回 vs 人間は数百回）。

### 考察

#### 「胃空 = 空腹MAX」は設計ミスだった

B-2では `hunger = 1 - stomach.contents / capacity` という直接対応だった。人間の空腹感は血糖値が下がるサイン。血管（バッファ層）を挟むことで：
- 食後は血糖値が維持されるため空腹感が低い（2〜3時間）
- 胃が空になってからではなく、血糖値が落ちてきたときに空腹になる

#### 定期授乳の廃止で因果関係が成立した

B-2の定期授乳は「腹が減った → 泣く → 授乳される」という因果関係を壊していた。B-3からは泣き→授乳のサイクルのみになり、「まんま」という音に意味が付く前提が整った。

#### 自発喃語が「発達段階の橋渡し」になる

B-2まで太郎は親に話しかけられた34回しか発声しなかった。B-3では喃語240回が加わり、発達段階（喃語 → オウム返し → 初語）に必要な練習量が確保された。

     - stage2（舌音追加）: 9ヶ月 = 23,328,000秒

### 設計メモ

#### 声道（VocalTract）の設計上の課題

`update_stage()` は `taro/body/vocal_tract.py` に定義されており、Taroの体オブジェクト上のメソッドとして存在している。しかし「声道を成熟させる」という能動的なメソッドをTaroが持つのは人間模倣として不自然（人間は意識的に声道を成熟させない）。

将来的には「声道は生まれついての制約として最初から正しい段階に入っている」設計が理想——つまり声道オブジェクトは `get_available_phonemes(sim_seconds)` のように時刻を受け取って現在使える音を返すだけにして、`update_stage()` のような「状態を書き換えるメソッド」は持たない。今は許容範囲として残す。

### 考察

#### 1シミュレーション秒 ≠ 1現実秒だった

B-4まで `stage1_time: 300` という設定は根拠のない仮置きだった。人間の規準喃語（ままま・ばばば）は生後6〜10ヶ月から始まる。1シミュレーション秒を1現実秒として設計すると、stage1 は180日×86400秒 = 15,552,000秒が正しい。

この修正により：
- 7日間シミュレーションでは母音のみ（クーイング期）が正しく再現される
- 初語（まんま）検証には最低9〜14ヶ月相当のシミュレーションが必要になる

8. **B-6：「ん」を声道に追加**

   #### 問題の発見
   - 「ん」は調音表に既に定義されていたが place=5（軟口蓋）→ stage3（12ヶ月）まで使えなかった
   - 「まんま」の「ん」は「ま」の前で両唇鼻音[m]に同化する → place=両唇(1)が正確

   #### 変更内容（vocal_tract.py）
   - VOWELS に index 5「（母音なし）」を追加。NUM_VOWEL: 5→6
   - 「ん」の定義: (5,1,1,0)→**(1,1,1,5)**（両唇・鼻音・有声・母音なし）
   - STAGE_ALLOWED_VOWEL: stage0は0-4のみ。**stage1以降に index 5（ん）を解禁**
   - 効果: 6ヶ月以降、喃語で「ん」を産出できる。「まんま」の音形が理論的に可能に

### 考察

#### 「ん」追加だけでは初語は出ない（必要条件・充足条件の分離）

「ん」を追加して「まんま」の音形が可能になった。しかし以下の課題が残る：

1. **発話長の問題**: 現在の喃語は10〜50文字。「まんま」は3文字。短い特定語を選ぶ仕組みがない
2. **意味結合の問題**: m音は6ヶ月以降すべての喃語に出るが、hunger=0でも出る（空腹との選択的結合が弱い）
3. **確率の問題**: ランダムな文字列の中にま+ん+まが偶然入るのを待つだけでは、「伝達的使用」の証明にならない

次のシミュレーション（B-6版）で「ん」の出現頻度と「まんま」の自発産出が起きるかを検証する。

2. **B-7：停止ヘッド実装・削除**

   - 「発話を途中で止める学習」をするため `head_stop = Linear(hidden_dim, 1)` を実装
   - 親の発話のEOS位置から停止を学習する設計
   - **結果**: 空喃語 69% 発生。原因は知覚時（親の音列を受け取る）と生成時（自分で出す）の隠れ状態が異なり、停止ヘッドが生成時に誤発火

3. **B-8：発話→授乳の順序修正**

   - 変更前：`env.feed()` → `env.step("まんま")` （満腹のまま聞く）
   - 変更後：`env.step("まんま")` → `env.feed()` （空腹のまま聞く）
   - 検証結果：まんまを聞くときの hunger 平均が 0.1 → **0.869** に改善

4. **原因調査：なぜ hunger→まんまの学習が起きないのか**

   コードを追うと、`brain.generate()` が `body_state` を受け取っていなかった。
   - 知覚学習（`learn_perception`）には body_state を渡していた ✓
   - 生成（`generate()`）の内部ループでは `forward_hidden(token, hidden)` を `body_state=None` で呼んでいた ✗
   - 発声の各ステップで Insula 出力が **常にゼロ** → GRU は hunger を参照できない

5. **B-9：`generate()` に `body_state` を追加**

   修正箇所3か所：
   - `cortex.py`：`generate()` のシグネチャに `body_state=None` を追加。BOS処理と自己聴取ループの両方で `forward_hidden(..., body_state=body_state)` を渡すよう変更
   - `core_b.py`：`step()` と `self_babble()` の `brain.generate()` 呼び出しに `body_state=body_state` を追加

### 考察

#### 知覚と生成で body_state が非対称だった

学習（知覚）時は hunger が GRU に届いていた。しかし生成時は届いていなかった。これにより GRU は「空腹+まんまを聞く → 次はんが来る」を学習しても、「空腹 → まんまを出す」を生成時に活用できなかった。B-9 でこの非対称を解消した。

#### B-7の教訓：停止学習は人間模倣が難しい部位

乳児が発話を止めるメカニズムは（1）呼吸制御、（2）ターンテイキングの社会的同期、（3）音節単位の運動プログラム終了 が複合している。どれも現在の太郎には未実装の高次機能。肺活量（morae 数）で長さを自然に制限する現在の設計が当面は最も人間模倣に近い。
