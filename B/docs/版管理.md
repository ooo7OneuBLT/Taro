## 目標B：初語（音に意味がつく）

| 版 | 変更内容 | 結果 | 状態 |
|----|---------|------|------|
| **B-1** | 身体シミュレーション＋脳の拡張。胃（消化→空腹）、肺（肺活量→発声モーラ数）、内受容感覚（つらさ1次元）、島皮質（体→脳への入力線）、恒常性の本能（つらさ低下→報酬）、泣き（段階的・確率的）、イベント駆動時間、親シミュレータ、手動CLI、スケジュール設定yaml | 基本ループは動作。1日シミュレーション：泣き70,297回（多すぎ）、睡眠62回（頻繁すぎ）。食事が一瞬で入る、泣きが毎秒判定、寝入りが即時という問題あり | ✅ → B-2で修正 |
| **B-2** | 行動を「一瞬」から「期間がある」に修正。授乳＝30分間、泣き＝数分間継続、寝入り＝うとうと5-10分→睡眠2-4時間。胃の消化速度を新生児に合わせて調整 | 1日シミュレーション：泣き43回、食事32回、睡眠6回。人間の新生児に近づいた。オウム返しも出る（転移テスト0.90-0.99） | ✅ → B-3で修正 |
| **B-3** | 血管（血中栄養素）を追加し「胃空=空腹MAX」を解消。定期授乳を廃止し「泣いて空腹のときだけ授乳」に変更。自発的な喃語（脳の現在の分布からサンプリング）を実装 | 1日シミュレーション：泣き35回、食事13回、喃語240回、睡眠6回。全指標が人間の新生児の範囲に | ✅ |
| **B-4** | アデノシン状態変数を追加し眠さをメカニズムから算出（`adenosine.py`）。数式決め打ちを廃止 | 1日：泣き50回、食事13回、喃語381回、睡眠6回。全指標が人間範囲内 | ✅ |
| **B-5** | 海馬バッファ実装（`hippocampus.py`）。喃語経験を覚醒中に蓄積し睡眠移行時に大脳皮質へ定着（McClelland 1995 CLS理論）。逆伝播を330回/日→6回/日に削減 | 9ヶ月57分。まんま自発産出0件（「ん」が語彙にない）。r_imit が6ヶ月でジャンプ（0.41→0.66） | ✅ |
| **B-6** | 「ん」を両唇鼻音(1,1,1,5)として定義しstage1(6ヶ月)から解禁。肺活量バグ修正（`stamina=0`が偽値判定→50字喃語）。最大モーラ数15→8・成長速度調整で喃語長を人間に合わせた | 9ヶ月19分（57分→大幅短縮）。まんまオウム返し97件。まんま自発産出0件 | ✅ |
| **B-7** | 停止ヘッド（`head_stop`）を実装し、発話の長さを学習で制御しようとした | 空喃語69%発生。原因：知覚時と生成時で隠れ状態が異なるため停止ヘッドが誤発火。人間模倣の根拠も弱いと判断し削除 | ❌ 削除 |
| **B-8** | 親の発話→授乳の順序を逆に（「まんま」を言ってから授乳）。空腹のまま「まんま」を聞けるよう修正 | 9ヶ月19.6分。まんまオウム返し73件。まんま自発産出0→**213件**に増加。hunger相関はまだなし（差-0.009） | ✅ |
| **B-9** | `generate()` に `body_state` を渡すよう修正。発声の各ステップでhunger等がGRUに届くようにした（B-8まではゼロが入力されていた） | 未検証（次回シミュレーション） | 🔄 現行版 |

### 残課題

- hunger→まんまの相関がまだ弱い（B-9で修正済み、検証待ち）
- まんまの自発産出が「空腹のとき限定」にならない（意味の結合が弱い）
