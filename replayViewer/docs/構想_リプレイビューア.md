# 構想メモ：太郎リプレイビューア

太郎のシミュレーション中に「体のどこが発火し、情報がどう流れ、数値がどう動いたか」を
動画プレイヤーのように再生・巻き戻し・場面ジャンプして見るためのツール。

**位置づけ**：AIモデル本体（`B/`）とは別物。ここ `replayViewer/` 配下で開発する。
**着手タイミング**：目標Bの「理解テスト」が実装・成功したあと（それまでは構想のみ）。
作成：2026-07-02。

**追記（2026-07-07）**：以下は着手前の初期構想であり、実装は複数回のバグ修正を経て
方針が変わった点がある。現在の実際の挙動：
- 「④(b)イベント」の`modules`/`flows`（部品IDと矢印を決め打ちで書く方式）は廃止。
  各部品の発火は`_organ_activity()`でシミュレーションの実測値から算出し、ネットワーク図の
  エッジも「両端が実測で発火しているか」で光らせる（固定配線×実測点灯）。
- 体ビューの色分けは単一色相（赤）の濃淡グラデーションに変更（虹色ヒートマップは
  見づらいとして撤回）。
- 「継続する状態」（消化中・睡眠中のバッファ残量等）と「一時的な活動」を区別し、
  臓器は後者のみで点灯する基準を導入（詳細：[研究日誌.md](../../B/docs/研究日誌.md)の
  2026-07-05以降）。

---

## 1. 目的

シミュレーションが裏で走っていると中で何が起きているか分かりづらい。
ログを入力すると、視覚的なモデルとして「働いている部品の強調表示・情報の流れ・
数値の推移」を、リアルタイムでなくてよいので後から再生して確認できるようにする。

---

## 2. 画面レイアウト（著者のスケッチに基づく）

```
┌───────────────────────────┬───────────────────────────┐
│  ①体ビュー（2D人体の模式図）      │  ②ネットワーク図（抽象グラフ）    │
│   ・脳（大脳皮質・小脳・青斑核…）  │   ・各部品をノードで表示        │
│   ・胃・肺・声道 など           │   ・発火したノード/エッジを赤く   │
│   ・働いている部品だけ強調表示     │    「発火!」表示、流れを矢印で    │
│   ・情報の流れを矢印アニメ        │                           │
├───────────────────────────┴───────────────────────────┤
│  ③数値一覧（バー/ゲージ）                                   │
│   ・ドーパミン分泌量  ・幸福度  ・空腹 ・眠さ ・NE(探索) など    │
├──────────────────────────────────────────────────────┤
│  ④タイムライン（0年 ──●(再生) ── 2年）                       │
│   ・再生ヘッドが日付を指す（例：3か月15日）                    │
│   ・再生/停止・巻き戻し・場面ジャンプ・速度                     │
└──────────────────────────────────────────────────────┘
```

- ①体ビューと②ネットワーク図は**同じ発火情報を2つの見方で並べて表示**する
  （体の中の場所 と 抽象的なつながり、の両方）。
- ③はゆっくり動く数値のゲージ。
- ④は動画プレイヤーのシークバー。0年〜2年を日付でスクラブできる。

## 4. トレースログの形（案）

別ファイル `trace.jsonl` に2種類の行を吐く。記録するのは**小さな数値と部品IDだけ**
（GRU隠れ状態などの重いテンソルは保存しない＝速度・容量のため）。

### (a) スナップショット（1時間ごと。ゆっくり数値の推移用）
```json
{ "type":"snap", "t":9720000, "date":"0y3m15d",
  "hunger":0.42, "sleepiness":0.3, "discomfort":0.1, "arousal":0.42,
  "ne":0.55, "dopamine":0.12, "happiness":0.58, "stage":2 }
```
- `happiness`(幸福度) と `dopamine`(分泌量) は**表示用に定義が要る**（下記「未決事項」）。

### (b) イベント（発火のたび。体ビュー/ネットワーク図の強調用）
```json
{ "type":"event", "t":18100000, "date":"0y7m0d", "kind":"word_request",
  "modules":["stomach","insula","cortex","vocal"],
  "flows":[["stomach","insula"],["insula","cortex"],["cortex","vocal"]],
  "utter":"まんま", "hunger":0.72,
  "rewards":{"r_mand":0.21} }
```
- `kind`：babble / babble_response / word_request / feed / comfort / cry_start /
  cry_end / sleep_start / sleep_end / consolidate など。
- `modules`：その瞬間に働いた部品のID（安定した固定ID）。
- `flows`：情報が流れた部品ペア（矢印アニメ用）。

### 部品ID（体ビュー/ネットワーク図で共通に使う）
cortex(大脳皮質) / cerebellum(小脳) / locus(青斑核) / critic(クリティック) /
insula(島皮質) / hippocampus(海馬) / basal_ganglia(基底核) /
stomach(胃) / lungs(肺) / vocal(声道)

## 5. 段階（着手順）

1. **トレースログを足す**（`B/` 側。ここが土台。フラグでON/OFF、別ファイル出力）
   - メインループで1時間境界を跨いだらスナップショット。
   - self_babble / respond_to_babble / step / feed / cry / sleep / consolidate の
     各所で、働いた部品IDと流れを event として記録。
2. **2D静止ビュー**：trace.jsonl を読み、ある1コマ（体＋光る部品＋数値）を表示。
3. **プレイヤー化**：再生/停止/速度/シークバー（日付）/場面ジャンプ。
4. **ネットワーク図**：抽象グラフで発火ノード/エッジを赤表示＋流れアニメ。
5. **数値ゲージ**：ドーパミン・幸福度・空腹などの時系列バー。
6. （応用）3D化 / リアルタイム追従。

## 6. 性能の見積もり（重くなりすぎないか）

- 1時間スナップショット：2年で17,520行 → 既存の喃語ログ(年10万行)に比べ誤差。
- イベント：2年で約20万件。書き込みはバッファ済み。既存イベントログに1行足す程度。
- 記録は小さなスカラー/IDのみ、テンソルは保存しない。
- フラグOFFで通常ランは完全に無コスト。
→ 想定オーバーヘッドは数%以内。実用上問題なし。

## 7. 未決事項（着手時に決める）

- **「幸福度」の定義**：太郎に幸福度という変数は無い。表示用に定義する必要がある。
  候補：`1 - arousal`（つらさの裏返し）／ 恒常性報酬 r_home の移動平均（ホッとした度）。
- **「ドーパミン分泌量」の定義**：dopamine の RPE か baseline のどちらを出すか。
- スナップショット粒度は1時間で始め、重い/粗いと感じたら日/30分に調整。
- 体ビューの人体イラストの描き方（SVGで手描き風か、シンプルな幾何模式図か）。
- ネットワーク図のレイアウト（固定配置か力学配置か）。
