## 全体設計：環境とエージェントの分離

強化学習の基本構造に沿って、太郎の脳を「エージェント」、身体・世界を「環境」として分離する。

```
環境（決めない側）
  = MIMo（手足・視覚・触覚・固有感覚・前庭覚・物理身体）
  + 親シミュレータ（発話・授乳・世話）
  + 太郎の内受容感覚（空腹）・発声・聴覚モジュール（MIMoに無いため持ち込む）

エージェント（決める側）
  = 太郎の脳（GRU cortex + 本能群）── ほぼそのまま流用
```

身体はエージェントではなく環境側にあるため、太郎の脳を無傷のまま別の身体に繋ぎ替えられる、という前提に立つ。

なお、MIMoと太郎の身体は**置き換えではなく補い合う関係**である。MIMoは手足・目・皮膚（運動・知覚系）を持つが、胃・空腹・声（内臓系）は持たない。逆に太郎は内臓系は持つが手足を持たない。そこで、MIMoに無い内受容感覚などは太郎から持ち込み、ラッパー層でMIMoと束ねて1つの身体として脳に見せる（＝ハイブリッド環境）。

## MIMoの5つの感覚（Phase 3で中身を確認）

観測・行動の次元だけでなく、各モジュールの実装を読み、何を模しているかを確認した。

|感覚|何をしているか|
|-|-|
|**視覚**|ただのカメラではない。月齢が低いほど画像をぼかす処理（実測データに基づく、乳児の視力の未発達さを再現）と、画面の中心だけくっきり・周辺はぼやける処理（人間の目の中心窩の構造そのもの）が、太郎に渡す前段階で既にかかっている|
|**触覚**|全身に4,274か所のセンサーが貼られており、各センサーが「押される強さ」と「こすれる方向」（3つの数字）を返す。4,274×3＝観測の12,822次元と一致|
|**固有感覚**|関節93か所について、角度・動く速さ・かかっている力・曲げすぎ/伸ばしすぎでないか、の4種の情報が来る。関節93に対し実際に駆動される関節（アクチュエータ）は90＝3関節は受動的（連動して動くだけ）|
|**前庭覚**|頭の中の傾きセンサーと揺れ（加速度）センサー、計6個の数字|
|**運動の出し方**|3通りの方式が用意されている：①関節をバネのように扱い力の入れ具合を指定する方式（既定）②関節角度を直接指定して固定する人形操作的な方式（ポーズ再現用）③向かい合う2本の筋肉が引っ張り合う、より本物に近い筋肉方式|

視覚が「生のピクセルではなく、既に人間の目らしく加工された画像」である点は、下記の論点1（CNNの位置づけ）を考える上での土台になる。

## 検討中の設計論点

技術的な障害（環境構築・視覚レンダリング・計算コスト）はPhase 0-2で解消済み。残る論点は太郎の哲学（生得の本能だけを与え、答えを注入しない）との整合性に関わるもの：

1. **視覚エンコーダ（CNN）の位置づけ**：MIMoの視覚は（上記の通り年齢依存のぼかし・中心窩処理を経ているとはいえ）依然として画像そのもので、太郎の脳（GRU）にそのまま入力できない。間に置く圧縮層（CNN）を、「視覚野に相当する生得の前処理」とみなせるか、それとも人間が持たない機能の注入にあたるか。
   - **暫定決定（2026-07-10）**：CNNは【既存AI研究】⚠️逸脱として正直にラベルを貼った上で採用し、実装を進める（GRU＋backpropも同様に既存AI研究として使っている前例に準拠）。より生物学的に忠実な代替（学習型CNNでなく網膜/V1の固定受容野を模した固定フィルタ、あるいはGRU自体をスパイキングニューラルネット＋STDPに置き換える等）は、今の実装を作り直す規模になり、既存の学習結果を失うリスクが大きいため、**今回のスコープでは採用しない**。どうしても生物学的忠実性を追求したくなった場合は、実績・知見が蓄積した状態から**別プロジェクトとしてやり直す**方が、ゼロからやるより現実的という判断。
2. **多感覚性 と 感覚運動予測ループの関係**：視覚・触覚などの多感覚そのものと、自分の運動の結果を予測して誤差を修正する仕組み（感覚運動予測ループ）は、どちらがより本質的か、あるいは不可分か。
3. **運動出力の設計**：発声4パラメータから全身90関節への拡張時、上記3方式（力任せ制御・位置固定・筋肉方式）のどれが妥当か。
4. **内受容感覚・聴覚の合成方法**：MIMoに無いこれらを、どのようにハイブリッド環境に組み込むのが自然か。

これらはPhase 4以降の実装着手前に方針を固める。
