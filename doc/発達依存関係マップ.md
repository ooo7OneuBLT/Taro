# 発達依存関係マップ（太郎）

太郎の各能力が**「何の上に立つか（依存関係）」**を、発達神経科学・発達心理学の**確立した枠組みで裏どり**して整理する。目的＝**目標の順番を"実用的な思いつき"でなく"発達の依存関係"で決める**（＝人間模倣ファーストの徹底）。[運動発達ロードマップ.md](運動発達ロードマップ.md)の依存関係版。

## 大枠：2つの並行トラック＋自己→他者の軸
- **運動トラック**：運動性喃語 → 自己モデル → 目的的運動
- **音声トラック**：喃語 → オウム返し → 初語
- **社会軸**：自己モデル →（投影）→ 他者理解
- 運動トラックと音声トラックは**並行**（赤ちゃんは同時にやる）。**厳密な順番が要るのは軸内の依存だけ**。

## 統合タイムライン（月齢は標準的な目安）

| 現象 | 太郎の目標 | 月齢目安 | 何の上に立つ（依存） | 裏どり文献 |
|---|---|---|---|---|
| **運動性喃語**（自発運動・writhing） | C5 | 0-2m | 出発点＝**一次変動性**（外界に合わせない自発的な運動の変異） | Hadders-Algra NGST |

## 依存グラフ（矢印＝"これが先に要る"）
```
運動性喃語(一次変動性 0-2m)
   │ 自発運動＋感覚で較正
   ▼
自己モデル(agency・順モデル) ──────────────┐
   │ 感覚で最適運動を選択(二次変動性)         │ 遠心性コピー/前庭で自己運動を割り引く
   ▼                                        ▼
運動性喃語の終了→目的的運動・リーチ(5m)   自己運動の割り引き(egomotion)
                                             │  ＋自己モデル("like me")
                                             ▼
                                        他者理解(D, 9-12m+)

[並行] 喃語(原音2-3m→canonical 7m) → オウム返し(模倣) → 初語(12m)
```

## 厳密に守るべき依存（文献の根拠つき）
1. **自己モデル → 他者理解**：他者は"自分と同じ(like me)"と見なし、**自分の内的状態を他者に投影**して理解する。＝自己が先。（Meltzoff 2007）
2. **運動性喃語(一次変動性) → 目的的運動(二次変動性)**：まず外界に合わせない自発的変異、次に**感覚情報で最適を選択**する適応段階。順番が決まっている。（Hadders-Algra 2000/2018）
3. **自己運動の割り引き → 動く自分での視覚的他者理解(egomotion)**：voluntaryな自己運動は**遠心性コピー＋前庭＋固有感覚**で推定・割り引く必要。＝自己モデルの視覚への拡張が先。（前庭-自己運動統合）
4. **canonical喃語 → 初語**：初語はcanonical喃語を土台に約1歳で出る。（Oller）

## 並行でOK（順不同）
- **運動トラック ↔ 音声トラック**：同時並行で発達する。どちらを先に扱っても発達的に破綻しない。

## 太郎の実際の順番の評価
太郎：**A(オウム返し)→B(喃語→初語)→C(自己モデル)→D(他者理解)**、C5(運動性喃語)は最近。
- ✅ **合っていた**：C(自己モデル)→D(他者理解)＝Meltzoff通り
- ✅ **問題ない**：音声(A,B)を運動(C,C5)より先に＝並行トラックなのでOK
- ⚠️ **やや前後**：自己モデル(C)を先に確立し、運動性喃語の"質"(C5)を後で。ただし自己モデルはCの自発運動で形成済＝実害小
- ❌ **依存を破った（唯一"効いた"順番違反）**：**運動性喃語が終わる前＋自己運動割り引き機構が未実装のまま、Dの"動く自分での視覚的他者理解(egomotion)"をやった** → 壁の原因（依存3を飛ばした）

## 結論・今後の指針
- **全部を一列にやる必要はない**（音声と運動は並行）。**厳密なのは軸内依存（上の1〜4）だけ**。
- **次の正しい順番**：
  1. **運動性喃語の終了（一次→二次変動性の移行＝目的的運動へ）**を作る
  2. **自己モデルを視覚へ拡張**（遠心性コピー/前庭でegomotionを割り引く機構を、門番でなく太郎の脳に実装）
  3. **その後にDの"動く自分での他者理解"**
- ＝Dの壁は「間違った順番」の帰結。この依存マップに従えば回避できる。
