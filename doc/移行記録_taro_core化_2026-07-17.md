# 移行記録：太郎の脳・paths・2体環境を `taro_core/` へ切り出し（2026-07-17）

**目的**：太郎の脳（`C/src`）・場所案内板（`C/paths.py`）・2体環境（`D/scripts/d_env.py`等）が
目標フォルダ `C/`・`D/` の中に埋まっていて、D が隣の C に `sys.path` で手を伸ばして借りている
歪んだ依存を解消する。太郎の脳は「目標Cのもの」でなく「太郎そのもの」なので、中立な
`taro_core/` に置き、C も D も対等に import する形にする。

**背景の判断**：
- 脳を外に出す＝根本を直す（ユーザー承認）。
- 2体環境も外に出す＝最終ゴールが介護AI（相手がいる世界）で、C以降はほぼ全部他者を扱う
  ＝2体環境は共有インフラだから（ユーザー承認）。
- MIMo（体）は既に `Taro/MIMo` に独立済みで、今回は触らない。
- A/B は独自の脳コピーを持ち C 非依存なので影響なし（今回は触らない）。

**安全策**：作業前の作業ツリーはクリーン（全コミット済）。各ステップを「動く状態」で
こまめにコミットするので、`git log` とこの記録を突き合わせれば巻き戻し先が分かる。
壊れたら `git reset --hard <その手前のコミット>` で戻せる。

## 依存構造の事実（移行が安全な根拠）

- `C/src` は**パッケージでなくフラット構成**（`__init__.py` 無し）。モジュールは**素の名前**で
  import し合う（例：`from hippocampus import ...`）。"C" という文字は import に埋め込まれていない。
  → ディレクトリごと動かせば 22 モジュール本体は**無修正**で動く。
- パスを通しているのは2箇所だけ：
  1. `paths.setup_brain_path()` が `src/{wrapper,senses,brain}` と `tests` を sys.path に載せる。
  2. `src/wrapper/hybrid_env.py` が自分の隣の `../body` を `__file__` 基準で載せる（相対なので移動に強い）。
- 外部（D等）が使う paths の窓口は **`SCENE` / `MIMO_DIR` / `setup_brain_path()` の3つだけ**
  （`paths.SRC/C_ROOT/LOGS/TESTS` を外部から使う箇所は grep でゼロ）。
- テスト（`C/tests/*.py`）は `__file__` 基準で自分の `../src/wrapper` を載せる自己完結型
  → コピー先でも無修正で動く。
