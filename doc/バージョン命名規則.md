# バージョン命名規則

太郎（Taro）のバージョンは **4層＋指標注釈** で表す。ソフト業界の semver（メジャー.マイナー.パッチ）に対応する。

## 形

```
太郎 C 2 - 13 (G1)
 │   │ │   │   └ 指標注釈（その版で測る指標。任意）
 │   │ │   └──── パッチ：プッシュのたび +1
 │   │ └──────── マイナー：目標内の能力マイルストーン
 │   └────────── メジャー（英字＝発達目標）
 └────────────── 世代（名前）
```

`太郎C2-13` ＝「太郎世代・目標C・2番目のマイルストーン・13回目のプッシュ」。

## 各層の意味

| 層 | 意味 | 例 | semver |
|---|---|---|---|
| 世代（名前） | 根本的な作り直し（アーキテクチャ刷新） | 太郎 → 次郎 → 三郎 | 製品名 |
| 英字（＝目標） | 発達目標。下表に固定 | A / B / C / D | MAJOR |
| 数字（＝節目） | 目標内の能力マイルストーン | C1 / C2 | MINOR |
| -数字（＝プッシュ） | 実装イテレーション | C2-1 / C2-13 | PATCH |
| (括弧)＝指標 | その版で測る指標の注釈 | 太郎C2-13(G1) | ラベル |

### 英字＝発達目標（固定）

| 目標 | 内容 |
|---|---|
| A | オウム返し（親の発話の模倣） |
| B | 自発的に意味のある言葉を話す（初語） |
| C | 自己内部モデルの確立（自分の体の順モデル） |
| D | 内部モデルを他者に当てはめる（他者理解） |

### 指標注釈（目標Cの合格条件など）

- G1：自己予測が本物か（入替マージン＋変化相関）
- G2：agency（自分が動かした vs 外から動かされた）

目標レベルなら `太郎C(G1)`、特定版の結果なら `太郎C2-1(G1)`。

## 昇格ルール

- **パッチ -N**：プッシュのたび +1。上位が上がったら 1 に戻す。
- **マイナー N**：新しい仕組み／能力の節目で +1、パッチを 1 に。（バグ修正・微調整はパッチのみ）
- **メジャー 英字**：目標を達成して次の目標へ。数字を 1 に。
- **世代 名前**：作り直しで 太郎 → 次郎。以下リセット。

目標をまたいで作業した版は、**主に前進させた目標の英字**を採る。

## 表示形と機械形

- 表示（人間向け）：`太郎C2-13`
- タグ・ファイル・機械向け：ASCII で `taro-C2-13`（次郎=`jiro`, 三郎=`saburo`）。git タグや自動処理で漢字の不具合を避けるため。

現在の版はリポジトリ直下の `VERSION` に機械形で記録する。

## プッシュ運用（＝バージョン確定の手順）

プッシュするたびにパッチ番号を上げる。手順を固定：

1. 公開ファイルに非公開情報（個人情報・私的な文脈など）が混じっていないか確認し、あれば除去する。
2. `VERSION` のパッチ番号を +1（新しい仕組みならマイナーを +1 してパッチを 1 に）。
3. コミット → プッシュ → `git tag <機械形>`（例 `git tag taro-C2-1`）。

## 履歴の起点

- これまでの目標C（順モデル基礎、G1・G2 達成）＝ **太郎C1**。パッチ未計測のため、現在のコミット済み状態を **太郎C1-1** として起点とする。
- 記憶定着（睡眠リプレイ）の本実装＝新しい仕組みの節目 → **太郎C2**。最初のプッシュが **太郎C2-1**。
