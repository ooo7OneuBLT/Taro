## 実験・検証

- [ ] **予測幅を伸ばす方向の確認（K=200/500）** ＝"山型のピークがどこか"を見る
  - 背景：K=10〜100のスイープで「長いほど良い」だった。逆にもっと伸ばすとどこかで反転するはず（カオス支配）。
  - ⚠️注意：①**長Kで交絡が悪化**（発達そろえ→経験過多／経験そろえ→学習不足）②**伸ばすほど非人間的**（人間の予測は0.1〜0.2秒＋閉ループ）。
  - 位置づけ：**結論（K=100採用）を覆す目的でなく、曲線の形の確認だけ**。→ 研究日誌「予測幅スイープ」参照。

- [進行中] **閉ループ（フィードバック）で短い予測幅を再評価**
  - 人間の"短予測幅でも高精度"は**閉ループ（感覚で常時補正）**由来。開ループのまま短くしたらダメだった（実証済）。
  - 短K(K=10=0.1秒＝人間のフィードバック遅延)＋閉ループreaching訓練(held goal＋停止条件)を実装・検証中。→ 研究日誌「C4」。

- [ ] **短K学習の"信号不足"を効かせる候補（要検証）**
  - 問題：**0.1秒で"学習"すること自体が難所**。人間は一生かけて（長いスケールで）学び、0.1秒では**使うだけ＋時間積分＋多感覚融合**でノイズを消してる。太郎は0.1秒の窓から学ぼうとして信号がノイズに埋もれる。
  - 候補：①**学習と制御の時間スケールを分ける**（良い信号の長い窓で学び、短い間隔で制御に使う＝**階層的時間スケール**, Tani系 MTRNN の速い層/遅い層）②**時間積分**（カルマン/ベイズで、ノイジーな短窓の証拠を積んで推定を安定化）③**多感覚融合**（触覚等を足して信号を底上げ、今は固有感覚のみ）。
  - → 今の短K閉ループ訓練が**失敗したら**、これらで救えるか検証。成功しても①〜③でさらに人間的に。

## 公開・権利まわり

- [ ] **LICENSEを決める（保留・急がない）**
  - 現状：**LICENSEファイル無し**＝法的には「全権利留保」＝**他人はコードを一切使えない**（読むだけ）。
  - **急がなくてよい理由**：今は誰かが使いたがっている状況ではない。そして**厳しい方から緩い方へは動けるが、逆は動けない**（一度MITで配った版は永久にMIT。撤回不可）。＝迷っているうちは付けない方が安全。
  - **決めるときの材料**（2026-07-15に調査済み。再調査不要）：

    | | MIT | Apache-2.0 |
    |---|---|---|
    | 商用含め自由に使える | ○ | ○ |
    | 著作権表示を残す義務 | ○ | ○ |
    | **特許の許可が明文化** | **× 書いていない** | **○ 書いてある＋反撃条項**（このコードで特許訴訟を起こすと自分の使用許可が失効） |
    | **改変したら「変えた」と明記する義務** | × | **○** |
    | 長さ | 20行 | 200行 |
    | 例 | React, jQuery, **MIMo** | Android, Kubernetes, TensorFlow |

  - **Apache-2.0が有力な理由**：①介護AI＝実用を目指すので**特許条項が効く場面が来うる** ②「どこを変えたか明記する義務」が**太郎のオリジナルの取り分を守る** ③MITとほぼ同じ自由度なので研究として広がる道を閉じない。MIMoはMITだが、**Apache-2.0はMITのコードを取り込めるので矛盾しない**（緩い方は厳しい方に入れられる）。
  - **⚠️ライセンスとは別に注意（法律の専門家に要確認）**：**公開そのものが将来の特許を潰す可能性がある**。特許は「新しいこと」に与えられるので、自分で先に公開すると「もう世に出ている」ことになりうる（日本には新規性喪失の例外＝救済制度があるが、手続きと期限がある）。**ライセンスの選択と無関係に、公開した時点で発生する話**。介護AIを実用に持っていくなら、どこかの段階で詳しい人に確認する。

- [ ] **文献まわりの3ファイルの棲み分けを維持する**
  - `引用.md`＝**実際に使った研究だけ**（研究の中身が実装に入っているもの）。書誌は一次情報で裏取り済み。
  - `関連研究.md`＝**独自に到達した仕組みと既存研究の対応**。これは事実でなく**主張**なので引用と混ぜない。
  - `doc/参考文献リスト.md`＝**読んだ・当たった資料**のリソース集（リンク・書籍）。
  - ⚠️新しく文献を足すとき、**どれに入れるかを必ず判断する**。基準は「使ったか／対応か／読んだだけか」。
