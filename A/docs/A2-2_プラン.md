# A2-2 プラン：声道シミュレータ（太郎に口を与える）

## Context

A2-1の転移テストで、太郎は訓練フレーズを丸暗記しているだけで、
未見フレーズへの汎化ができないことが判明した（全滅・r_imit=0.00）。

原因：太郎にとって文字は「番号」であり、「ま」と「ぱ」に構造的な関係がない。
人間の赤ちゃんは口・舌・喉という身体を持ち、「ま」と「ぱ」は口の動かし方が
似ているため、一方を覚えれば他方も出しやすい。

A2-2では太郎に「口」（声道シミュレータ）を与え、文字を番号ではなく
「口の動かし方の組み合わせ」として扱うことで、汎化的な模倣を目指す。

## 変更内容

### 1. 声道シミュレータ（vocal_tract.py）新規作成

太郎の「身体」。4つのパラメータの組み合わせで文字を生成する。

パラメータ：
- 調音点（どこで音を作るか）：両唇/歯茎/歯茎硬口蓋/硬口蓋/軟口蓋/声門 + なし（母音のみ）
- 調音法（どうやって音を作るか）：鼻音/破裂音/摩擦音/破擦音/弾き音/半母音 + なし（母音のみ）
- 声帯（声帯を震わせるか）：有声/無声
- 母音（口の形）：あ/い/う/え/お

### 2. 脳の出力層を変更（brain.py）

変更前：文字の数だけの出力ノード（1つ選ぶ）
変更後：4つのパラメータそれぞれの選択肢を出力

### 3. 自己聴取の追加（environment.py）

太郎が出した音を自分で聞く（出力を入力に戻す）。
赤ちゃんは自分の声を聞いている【人間模倣】。

### 4. 音韻距離表は不要に

声道の構造から「ま」と「ぱ」の近さが自然に生まれるため、
親に音韻距離表を別途持たせる必要がなくなる。

## 検証方法

1. 訓練フレーズで学習が成功するか（A2-1と同等以上）
2. **転移テスト**：未見フレーズでr_imit>0になるか（A2-1では全滅だった）
3. 「ま」を覚えた後に「ぱ」が早く覚えられるか（部品の共有効果）
