人間のように育てるAI「太郎」
研究の道筋レポート

A1-1（最初の試作）から A2-5（発達段階の再設計）までの記録。
何を目指し、何を試し、何がうまくいき、何が壁になり、どう修正したか。

この研究を一言でいうと

「現在のAIとは違い、人間の赤ちゃんと同じやり方で言葉を覚えるAIは作れるか？」を確かめる個人研究。

最も大事なルール：太郎の中には人間の赤ちゃんが生まれつき持っている本能だけを入れる。人間に存在しない便利機能は使わない。壁にぶつかったら「人間の赤ちゃんと何が違うか？」を考え、その差を埋める。

各ステップの詳細

A1-1 — 最初の試作
「本能だけでオウム返しが生まれるか？」

目標：4つの本能（真似したい衝動・予測したい本能・ドーパミン・親の笑顔で喜ぶ）だけを太郎に与えて、親の言葉をそのまま返す「オウム返し」が自然に生まれるか確かめる。

予想：本能が正しく働けば、太郎は親の言葉を真似るようになるはず。

やったこと：「あ」「い」「う」を1文字ずつ交互に太郎に聞かせた。

「あ」1文字だけなら 93ターン（sim 7.8分）でオウム返しに成功！
3文字交互では2000ターンでも失敗。太郎は「い」だけを繰り返すようになった（固着）。

考察：1文字なら解けるが、複数文字の使い分けができない。そもそも「あ」「い」「う」を1文字ずつ出すのは、人間の親がやることではない。実際の親は「まま」「ばぁ」のようにフレーズで話しかける。

A1-2 — 温度の調整
「出力のランダムさを減らせば安定するのでは？」

予想：太郎は正解を知っているが、出力のランダムさ（温度）が高すぎて時々間違える。温度を下げれば安定するはず。

やったこと：温度の減衰速度を上げた。

むしろ悪化（最大連続一致が4回→2回に）。ランダムさが減った結果、間違った答えに早く固着した。

教訓：表面的な対症療法は根本原因を見逃す。「なぜうまくいかないか」をちゃんと分析すべきだった。

A1-3 — 短期記憶の修正＋フレーズ化
「赤ちゃんは聞いた音の記憶を使って直後に真似る」

人間との差を分析：

  • 太郎は親の発話を聞いた後、その記憶を捨ててから発声していた → 何を言われたか覚えていない
  • 出題が1文字ずつなのは人間的でない → 「まま」「ばぁ」等のフレーズに変更

修正：親の発話を聞いた直後の脳の状態（短期記憶）を使って発声するようにした。

「まま」「ばぁ」「ないない」の3フレーズで 355ターン（sim 29.6分）で成功！

知見：「聞いた直後の記憶（エコイックメモリ）」は模倣に不可欠。これは赤ちゃんの脳に最初から備わっている配線であり、新しい機能の追加ではなく「あるべき接続が欠けていたバグの修正」。

A2-1 — 身体的制約の追加
「赤ちゃんには身体の制約がある」

問題：太郎は「こんにちはこんはちはんにちは」のように延々と喋り続ける。赤ちゃんの口・喉・肺は未熟で長く発声できないはず。

追加した6つの改善：発声体力の制限 ／ 類似度の改善 ／ 温度を練習量に連動 ／ 成功条件の見直し ／ 未見フレーズのテスト ／ 親の笑顔の自動化

5フレーズを 348ターンで達成。長文喃語も消えた。
しかし転移テスト（初めて聞くフレーズでの模倣）は全滅。太郎は5フレーズを丸暗記していただけで、「真似る力」は身についていなかった。

考察：太郎にとって「ま」と「ぱ」は完全に別のもの。人間の赤ちゃんは口の動かし方が似ている（＝身体が教えてくれる）が、太郎には身体がない。

A2-2 — 声道シミュレータの導入
「太郎に口を与える」

着想：人間の赤ちゃんは口・舌・喉を動かして音を作る。「ま」と「ぱ」は口の動かし方が似ているから、一方を覚えれば他方も出しやすい。太郎にも「口」を与えれば、汎化的な模倣ができるのでは？

やったこと：4つのパラメータ（口のどこで・どうやって・声帯を使うか・口の形）で文字を生成する「声道シミュレータ」を作った。

学習が全く進まない。4つのパラメータを同時にランダムに選ぶので、偶然正しい組み合わせに当たる確率がほぼゼロ。太郎は「げげげ」や「れれれ」に固着。

人間との差：赤ちゃんは最初から4つ全部を制御するのではない。まず顎の開閉（母音）だけ、そこから段階的に制御できる部位が増える。

A2-3 — 段階的な口の発達
「赤ちゃんの口は段階的に成熟する」

根拠：Frame-Content理論（MacNeilage & Davis）。赤ちゃんの声道は時間とともに使える部位が増える。これは練習の結果ではなく、身体の物理的な成熟。

Stage人間の時期使えるようになる部位出せる音
00-3か月母音のみ「あー」「うー」
14-6か月＋唇「ばばば」「まままま」
28-10か月＋舌「だだだ」「ななな」
310か月〜＋声帯の制御「ぱ」と「ば」の使い分け
Stage 0で母音のみが出力された（クーイングの再現に成功）。
しかしStage 1に入ると特定の子音（「い」や「ぴ」）に固着して先に進めない。
A2-4 — 馴化（飽き）の追加
「赤ちゃんは同じ音に飽きる」

人間との差：太郎は同じ音を出し続けても飽きない。人間の赤ちゃんは同じ刺激に対する反応が徐々に弱まる（馴化）。これは胎児にも確認される最も基本的な神経メカニズムで、ドーパミンとは別の仕組み。

やったこと：5つ目の本能として「馴化」を追加。同じ出力を繰り返すと報酬が下がり、新しい出力を試すと回復する。

Stage 0で母音のバリエーションが明らかに増えた（以前は「いいいい」に固着していた）。
Stage 1以降で新しい子音に再固着（「ぱぱぱ」→「にぢぢぢ」→「よよよ」）。馴化だけでは足りない。
A2-5 — 成功基準の再設計
「そもそも赤ちゃんはいつオウム返しをするのか？」

重要な気づき：ここまで全StageでWウム返しを成功基準にしていた。しかし人間の赤ちゃんは生後9〜12か月まで意図的な模倣（オウム返し）をしない。Stage 0-3はそれぞれ別の発達課題がある。

反省：「人間模倣ファースト」を掲げながら、成功基準を人間の発達と照合していなかった。機能は人間に合わせたのに、「いつ何ができるべきか」を合わせ忘れていた。

再定義した成功基準：

Stage名前成功基準結果
1クーイング3種類以上の母音を出せる20ターンで達成 ✅
2声の探索5種類以上の文字＋子音が出始める達成 ✅
3規準喃語子音＋母音の組み合わせが安定形式上は達成だが固着 ⚠️
4オウム返し親との類似度≥0.8が10回連続未達成（Stage 3が不十分）
Stage 1が20ターンで達成。以前は「失敗」と見なしていた母音の出力が、実は「クーイングの成功」だった。見方を変えたら成功していた。

この研究で得られた知見

1. 壁→分析→修正のサイクルが研究の方法論

毎回「壁にぶつかる → 人間の赤ちゃんと何が違うかを考える → その差を埋める」で進んだ。これ自体がこの研究のやり方として確立された。

壁は失敗ではなく「太郎と人間の差が可視化された瞬間」であり、最も重要なデータ。

2. 「簡単にできた」は疑うべき

A1で声道なしのオウム返しが簡単にできたのは罠だった。人間に近い構造にしたら途端にできなくなったのは「前のやり方が人間的でなかった」証拠。簡単すぎる成功は、前提が間違っている可能性がある。

3. 機能だけでなく「いつ」も人間に合わせる

太郎の内部機能を人間に合わせることには注意を払っていたが、「この発達段階でこの行動ができるべきか？」を確認していなかった。ゴールの設計にも人間模倣が必要。

4. 身体が知識を生む

太郎に声道（口）を入れることで、「ま」と「ぱ」が構造的に近い存在になった。人間の赤ちゃんは身体を持つことで、誰にも教わらずに「似た音」の概念を獲得する。テキストだけのAIにはこれがない。

5. 「飽き」は本能

馴化（同じ刺激に飽きる）は胎児にも見られる最も基本的な仕組み。ドーパミン（報酬予測誤差）とは別に存在し、探索を促す。飽きがなければ、太郎は最初にたまたま見つけた答えに永遠に固着する。

現在の太郎の構造

太郎（taro/）
├── 身体（body/）
│   ├── 声道（vocal_tract.py） ← 口・舌・喉の物理構造
│   └── 体力（stamina.py）     ← 口の筋肉の成熟度
└── 脳（brain/）
    ├── 基本構造（core.py）    ← 予測する再帰ネット（GRU）
    ├── 学習（learning.py）    ← ⚠️backpropを使用（逸脱）
    └── 本能（instincts/）
        ├── 模倣衝動           ← 親に似せたい
        ├── 予測               ← 次を予測したい
        ├── ドーパミン         ← 予想より良かった/悪かった
        ├── 馴化               ← 同じことに飽きる
        └── （社会的報酬は環境側）

環境（environment/）← 太郎の外側
├── 親シミュレータ   ← 話しかける＋笑顔
└── CLI              ← 人間が直接対話
