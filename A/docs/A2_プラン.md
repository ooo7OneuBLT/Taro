## A2-1で実装する変更（6項目）

### 1. 発声体力の導入【人間模倣・身体的制約】

**問題**：太郎は50文字まで無制限に出力でき、長文の喃語が出る。赤ちゃんは口・喉・肺が未熟で長く発声できない。

**変更内容**：
- 太郎に「体力（stamina）」パラメータを追加
- 生成中に1文字出すごとに体力が減る
- 体力が尽きたら発声終了（<EOS>を強制）
- 初期体力は少ない（1〜3文字分）
- 発達年齢（sim_clock）に応じて体力上限が徐々に増える
- これは本能ではなく**身体的制約**として実装（instincts.yamlではなくconfig.yamlで管理）

**変更ファイル**：`brain.py`（generate関数に体力制限）、`config.yaml`（体力の初期値・成長率）

### 2. 模倣報酬の類似度関数を改善【人間模倣】(Q1)

**問題**：位置ごとの文字完全一致のみ。「近い音を出せた」段階での部分報酬がなく、漸進的模倣が再現できない。

**変更内容**：
- 位置一致率 → **正規化edit distance（編集距離）**ベースに変更
  - edit distance = 「何文字の挿入・削除・置換で一致させられるか」
  - 正規化 = edit_distance / max(len(parent), len(taro)) を1から引く → [0, 1]
  - 「まーま」→「まま」は1文字削除だけなので類似度が高い
- 将来的に音韻特徴距離（「ま」と「な」は近い等）も検討するが、A2-1ではedit distanceで十分
- 音韻距離表は**親シミュレータ側**に置く（親の知識であり太郎の知識ではない）。実装はA2-2以降に回す

**変更ファイル**：`instincts.py`（compute_imitation_reward関数）

### 3. 温度τを練習量・成功体験に連動【人間模倣】(Q2)

**問題**：τが時間で一律減衰。成功しても失敗しても同じペースで下がる。人間は練習で上達する。

**変更内容**：
- 外部スケジュール（decay_rate × 毎ターン）を廃止
- 代わりに**模倣報酬r_imitの累積に応じてτを減衰**させる
  - 模倣が成功するほどτが早く下がる（＝上手く真似できるほど発声が安定する）
  - 失敗が続くとτはあまり下がらない（＝上手くいかないうちは探索を続ける）
- 具体的な式：`τ = τ_initial / (1 + α * cumulative_r_imit)` （αは設定可能）
- これにより温度減衰が⚠️逸脱 → 【人間模倣】に昇格

**変更ファイル**：`brain.py`（decay_temperatureの仕組み変更）、`config.yaml`（α等）、`environment.py`（累積r_imitの追跡）

### 4. 成功条件の見直し【研究改善】(Q7)

**問題**：「完全一致10回連続」は人間より厳しい。部分的に似ていても失敗扱い。

**変更内容**：
- 成功基準を**2段階**にする：
  - **基本達成**：edit distance類似度 ≥ 0.8 が10回連続（だいたい合ってる）
  - **完全達成**：完全一致が10回連続（今までの基準、参考記録として残す）
- ログに毎ターンの類似度を記録し、**部分一致スコアの推移グラフ**も出力

**変更ファイル**：`environment.py`（判定ロジック）、`logger.py`（類似度の追跡・グラフ追加）、`parent_sim.py`（成功判定）

### 5. 未見フレーズでの転移テスト【研究改善】(Q7)

**問題**：3フレーズの丸暗記と汎化的模倣能力を区別できない。

**変更内容**：
- 親シミュレータに**テストフェーズ**を追加
  - 学習フェーズ：通常の訓練フレーズで学習
  - テストフェーズ：学習に使っていない**未見フレーズ**を投入（重みは更新しない）
  - 未見フレーズでの類似度を記録 → 丸暗記 vs 汎化を判定
- 例：訓練=「まま」「ばぁ」「ないない」、テスト=「ぱぱ」「だっこ」

**変更ファイル**：`parent_sim.py`（テストフェーズ追加）

### 6. 笑顔（社会的報酬）の基準整備【人間模倣】

**問題**：笑顔の強さ（r_social）の基準が曖昧。人間の親には反応パターンがある。

**変更内容**：
- 親シミュレータに**自動笑顔ロジック**を追加：
  - 初めて似た音を出した → 高い笑顔（驚き・喜び）
  - 同じことを繰り返しできる → 笑顔が薄くなる（馴化）
  - 前回より改善した → また高い笑顔（進歩への反応）
  - 具体的には `r_social = f(今回のr_imit, 前回のr_imit, 過去N回の平均r_imit)`
- **音韻距離表（「な」と「ま」は近い等）はA2-2以降で追加**。A2-1ではedit distanceベースの類似度のみで笑顔を判定
- CLIでの人間入力時のガイドラインもヘルプに表示

**変更ファイル**：`parent_sim.py`（自動笑顔）、`cli.py`（ガイドライン表示）

## 検証方法

1. **基本達成テスト**：訓練フレーズで類似度≥0.8が10回連続
2. **完全達成テスト**：訓練フレーズで完全一致10回連続（参考）
3. **転移テスト**：未見フレーズでの類似度を測定
4. **発声体力の効果**：出力長の分布がA1と比べて短くなることを確認
5. **τの適応的減衰**：成功が多い太郎ほどτが早く下がることを確認
6. **学習曲線**：r_imit（edit distanceベース）の推移をプロット
7. **A1との比較**：同じフレーズセットで、A1-3 vs A2-1の学習速度・安定性を比較
