# 目標B プラン：初語 — 音に「意味」がつく

## Context

目標A（オウム返し）は完了した。太郎は親の発声を真似られるようになり、
未見フレーズへの転移も成功した（模倣スコア0.94、転移時の完全一致80-100%）。
丸暗記ではなく、声道の構造を使って汎化できている。

次の発達段階は**初語**。人間の赤ちゃんが10-14か月で迎える、
「音に意味がつく」段階を再現する。

---

## 0. シミュレーション解像度のルール

太郎プロジェクト全体に適用する、身体シミュレーションの粒度の判断基準。

### ルール：段階的精密化

```
① その部品の「入力と出力」を特定する
② 入力を変えたとき出力が人間と同じように変わるか確認する
③ 同じ → その解像度で十分
④ 違う → 中の仕組みを1段階だけ細かくする
⑤ ③に戻る
```

### 原則

- **数式でカーブを決め打ちしない**。メカニズムを作り、カーブは結果として出す
  （声道でルックアップテーブルを作ったのと同じ発想。声道で「音の波形」を
  数式で作らなかったのと同じ）
- **臓器を増やすのではなく、臓器間の接続を必要に応じて足す**
  （例：血管を作るのではなく、胃→眠さの接続を作る）
- **脳に渡す信号のパターンが変わる部分は精密にする。
  信号を作る裏側の化学は作らない**
- **最初から最高解像度で作らない**。合わなくなったら1段階ずつ上げる
  （声道もA2-2→A2-3→A2-6と段階的に解像度を上げた）

### 接続漏れチェック（新しい学習系・記憶系を足したとき）【B2-11の反省から】

太郎は「経験が発生したら自動で全ての記憶・学習系に届く共通バス」を持たず、
「どの経験が、どの部品に届くか」を呼び出し側のコードで1本ずつ手配線している
（＝「臓器を増やさず接続を足す」方針の裏返し）。人間の脳も万能バスではなく
経路特化なので、この手配線自体は人間模倣として妥当。ただし**足し忘れると
何も起きない**という脆さがある。実際B2-11で「親との会話が睡眠リプレイ（海馬）に
一度も記録されていなかった＝理解の学習だけ機会が1/9になっていた」漏れが起きた。

そこで、新しい学習系・記憶系（critic・satiety_head・海馬など）を追加したら、
**次を明示的に確認する**：

1. **経験の発生源の棚卸し**：今ある経験の入り口を全部挙げる
   （現状：`step()`＝親との会話 ／ `self_babble()`＝自発喃語 ／
   `respond_to_babble()`＝喃語への反応 ／ 泣き・授乳・睡眠 など）
2. **新しい部品に、どの発生源から届くべきか**を1つずつ判断する
   （例：満腹予期は「親の発話を聞く」step()から届くべき。自発喃語からは不要）
3. **既存の発生源すべてについて、接続の要否を明記**する
   （不要なら「不要」と書く。空白＝判断漏れ、にしない）
4. 睡眠リプレイ（海馬）に載せるべき経験か、その場1回だけの学習か
   （頻度の少ない経験＝反芻で機会を補うべきか）を判断する

これは§0のルール（①〜⑤）に足す「⑥ 既存の全経験発生源から新経路への接続漏れ確認」に相当する。技術的な"入力の一本化"より、この開発プロセス上のチェックで漏れを防ぐ方が、経路特化という人間の実態にも合う。

### モデルAでの是正事項

モデルAの体力（stamina.py）は `+0.001/ターン` の数式決め打ちで、
肺のメカニズムから出た数値ではない。**モデルBへの移行でメカニズムベースに修正する**
（肺活量・呼吸筋の発達 → 発声できる長さが結果として伸びる）。

## 1. 目標Bの定義

**「まんま」というただの音が、「空腹を満たすもの」を指すようになる。**

目標Aとの決定的な違い：

| | 目標A（オウム返し） | 目標B（初語） |
|---|---|---|
| 発話のきっかけ | 親が言ったから真似する | お腹がすいたから自分から言う |
| 発話の性質 | 受け身（聞いたら返す） | 能動（状況に応じて発する） |
| 音と何かの関係 | 音は音でしかない | 音が「体の状態」と結びつく |

**発する音自体は変わらない**。「まんま」という音はAでもBでも同じ。
変わるのは「なぜ言うのか」。これが「意味」。

## 2. 人間の発達的根拠 【人間模倣】

| 月齢 | 段階 | 内容 |
|---|---|---|
| 9-12 | オウム返し | 親の音を真似る（＝目標A・完了） |
| **10-14** | **初語** | **特定の音に意味がつく（＝目標B）** |
| 12-18 | 一語文 | 1単語で要求を表す（将来の目標C） |

初語の特徴（発達心理学）：
- **文脈に縛られる**：最初の「まんま」は空腹時・食べ物を見たときにしか出ない
- **自発的**：親が言っていなくても自分から言う（オウム返しと違う点）
- **連合学習で獲得**：音と状況が何度も同時に起きる → 脳が結びつける

参考：初語の多くは世話に関わる語（まんま＝食事、まま＝母）や
社会的な語（ばいばい）。これは「音と切実な状況の相関」が強いから。

**学術的な位置づけ（オウム返しの「次」であることの根拠）**：
人間の音声発達は「クーイング→喃語→音声模倣（オウム返し）→初語」と進む。
オウム返し（vocal imitation, 9-12か月）の**直後の発達段階が初語**
（first words / one-word stage / 一語期, 10-14か月）であることは、
発達心理学・言語獲得論で広く共有された標準的な発達順序。
- 一語期・命名の洞察（naming insight）：今井むつみ『ことばの発達の謎を解く』
- 共同注意（joint attention）が語意獲得の前提：Tomasello (2003)
- ※太郎は視覚を持たないため、共同注意の代わりに**内部状態の共有**
  （親が世話をしながら言葉を添える）で「言葉と意味」の相関を作る。
  これは「指さしの先のもの」ではなく「体の状態」を指示対象にするという、
  太郎なりの初語の入口（より詳しくは§9・壁3）。

## 3. 初語の認定基準（人間の発達研究と同じ4条件）

「意味がついた」は主観的で曖昧なので、**人間の発達研究が「真の初語」を認める基準**に
合わせて検証する（ユーザー方針：認定基準は人間と同じに）。

| 人間の基準 | 太郎での対応 |
|---|---|
| **自発的**（真似・促しでなく自分から） | 親が言っていないのに言う（オウム返しとの区別） |
| **文脈が一貫**（同じ状況・対象で使う） | 空腹のとき言い、非空腹では言わない（発話率に有意差） |
| **音形が安定**（毎回ほぼ同じ音。大人発音と完全一致でなくてよい） | 「まんま」が毎回ほぼ同じ形で出る |
| **伝達的**（誰かに向け、効果を持つ） | 言うと親が来る／ごはんが来る |

この4条件を**すべて満たして初めて**「意味がついた＝初語」と認定する。
- 参考：phonetically consistent forms / protowords（Vihman, Bates）、
  今井むつみ『ことばの発達の謎を解く』

### 受信側の検証（補助）

親が「まんま」と言った直後、太郎が「次にお腹が満たされる」と予期できているか
（予測本能の的中度で測る）。

## 4. 必要な新コンポーネント

人間の身体・脳の部品に対応させて追加する。

### 4-1. 身体シミュレーション（taro/body/）【人間模倣】

太郎の身体を、§0のルールに従ってメカニズムベースでシミュレーションする。
**原則：生まれた状態に存在しないものは、生まれた太郎に追加しない。**

#### 胃（taro/body/stomach.py）

声道が「4パラメータ→文字」だったように、胃は「容器＋消化→空腹度」。

| パラメータ | 意味 |
|---|---|
| capacity | 胃の容量（成長で増える） |
| contents | 現在の胃の中身の量 |
| digestion_rate | 消化速度 |

- 入力：食事イベント（量）→ contents が増える
- 内部処理：contents が digestion_rate で時間とともに減る
- 出力：`hunger = 1 - (contents / capacity)` → 脳へ
- カーブは数式で決め打ちしない。「胃が消化で空になる」メカニズムから結果として出る
- 「少し食べた」「2回に分けて食べた」等にも自然に対応

#### 臓器間の接続

独立だと人間と違う挙動になる場合は、臓器間を接続する。

| 接続 | 理由 |
|---|---|
| 胃（消化吸収）→ 眠さ | 食後に眠くなる（新生児の基本行動） |

血管や心臓という「臓器」は作らない。必要なのは接続（A→B）だけ。

#### 肺（stamina.pyの修正）

モデルAの体力（`+0.001/ターン`の数式）を、肺活量・呼吸筋の発達に基づく
メカニズムベースに修正。発声できる長さが結果として伸びるようにする。

#### 将来の追加基準

新しい臓器は「その臓器の状態が言葉や行動の選択に影響する場合」だけ追加。

| 臓器 | 言葉の学習に影響？ | 追加時期 |
|---|---|---|
| 膀胱 | 「おしっこ」を学ぶなら | 将来かも |
| 心臓 | 「ドキドキ」を学ぶなら（2-3歳以降） | 当面不要 |
| 血管 | 接続で代替可能 | 不要 |

### 4-2. 内受容感覚と感情の分化（taro/body/internal_state.py）【人間模倣】

身体のシミュレーションから出る信号を統合し、脳に渡す。

#### 生まれつき入れる状態（生得）

| 状態 | 範囲 | 意味 | 由来 |
|---|---|---|---|
| hunger（空腹） | 0〜1 | 快適値は0（満腹） | 胃のシミュレーションから |
| sleepiness（眠さ） | 0〜1 | 快適値は0 | 覚醒エネルギー消費＋胃からの接続 |
| discomfort（不快） | 0〜1 | 痛み・濡れ等 | イベント |
| **arousal（興奮／つらさ）** | 0〜1 | **上記の総合。1次元のみ** | 上の状態から統合 |

#### 感情の分化（Bridges 1932）

- **0か月：興奮（arousal）の1次元だけ**。快/不快の両極すら無い
- **~3か月：快/不快に分かれる**。relief（つらさが減る安心）の
  繰り返しで「良くなる方向＝快」が芽生える
- **6か月以降：個別感情（怒り・恐れ・喜び等）に枝分かれ**
- **分化は手で足さない**。成熟（時期の解放）×経験で自然に枝分かれ
  - 「空腹は食で・さみしさは親で直る」と違う対処を学ぶ圧力で
    脳内の表現が引き剝がされる
  - 実装詳細は**B3（複数状態を複数語で言い分ける）で詰める**
  - B0〜B2は「arousal 1次元＋hunger」で足りる

※ loneliness は新生児の分化した感情ではなく、親不在による不快として
discomfort で表現。必要なら後で分化。

### 4-3. 恒常性の本能（taro/brain/instincts/homeostasis.py）【人間模倣】

**生得的な本能**。意味学習を駆動する原動力。

- 人間の根拠：ホメオスタシス（Cannon, 1932）、動因低減説（Hull, 1943）
- 各状態の快適値（hunger=0, sleepiness=0 等）に向かって戻す力
  - 快適値から離れる（hunger上昇）→ arousalが上がる → つらい
  - 快適値に戻る（hunger低下）→ arousalが下がる → ほっとする → 報酬 r_home
  - `r_home ＝ arousalが下がった量（ほっとした分）`

### 4-4. 島皮質（taro/brain/insula.py）【人間模倣】

体の感覚を脳（大脳皮質）に伝える経路。

- 人間の根拠：島皮質は内受容感覚を大脳皮質に届ける中枢
- 実装：内部状態の数値をベクトルに変換し、大脳皮質のGRU入力に合流

```
今のA：  耳（親の発話）→ 大脳皮質 → 口
目標B：  耳（親の発話）→ 大脳皮質 → 口（発話・泣き）
            ↑
         島皮質 ← 身体（胃・肺など）
```

### 4-5. 自発的発声＝泣き【人間模倣】

**太郎に最初から実装される唯一の能動的な行動が「泣き」。**

発声の発達順：
**泣き（0か月・生得）→ クーイング（~3か月・快が芽生えてから）
→ 喃語（~6か月）→ オウム返し（9-12か月＝目標A）→ 初語（目標B）**

- 目標Aとの違い：Aには「つらさ」が無かったので最初からクーイングしていた。
  Bでは泣きが先にあり、穏やかな時間（arousal低）にクーイング以降が育つ
- 泣きは音素（ま・ば）の産出とは別の「一般的な声」として実装
- **閾値ではなく段階的**：arousalの高まりに応じて
  「ぐずる→むずがる→泣く→激しく泣く」と確率・強さが連続的に上がる
- **最初は反射→やがて道具に**：「泣く→親が来る→つらさ消える」を経験して、
  意図的に親を呼ぶ手段に変わる。これが初語の前身

## 5. 時間モデルの変更：イベント駆動シミュレーション

目標Aは「1ターン＝1やり取り＝5秒」で、親が常にいた。
目標Bでは**親がいない時間**が必要。

### しくみ

```
毎tick：  身体シミュレーション進行（胃の消化など＝軽い）
たまに：  親が来る → 言葉 → 世話 → 脳が動く（重い）
不快時：  太郎が泣く → 親が来る
```

- 重い計算（脳・学習）は「やり取り」「泣き」のときだけ
- 何も起きない時間は身体の更新のみ。**次のイベントまで飛ばす**（イベント駆動）

## 6. 親モデルの拡張（environment/）

学習の仕掛けは**太郎の中ではなく環境（親）に置く**原則を守る。

| 行動 | 内容 | 太郎への影響 |
|---|---|---|
| 在/不在 | スケジュールや確率で決まる | 不在だとdiscomfort上昇 |
| 随伴反応 | 太郎が泣いたら来る（即・必ずではない） | 「泣く→親が来る」を学べる |
| 世話 | ごはん・抱っこ | hunger・discomfort回復 |
| 文脈発話 | 世話のとき言葉を添える（食事時に「まんま」） | 言葉と状況の相関を作る |

親の随伴反応は「即・必ず」ではない。時間差や不在がある。
いつも一発で叶うと「泣けば解決」で止まり、言葉への動機が育ちにくい。
このループ（泣く→親→安心）は、後の（まんま→親→満腹）の原型。

### 追加要件

1. **手動モード＋自動モードの両対応**
   - 自動：親シミュレータがスケジュールどおりに世話・発話
   - 手動：ユーザー自身が親になって直接話しかける・世話する（cli.py拡張）
   - 太郎本体は手動/自動を区別しない
2. **スケジュール・イベントを設定ファイルで変更可能に**
   - 親の在/不在、食事間隔、放置時間等を実験ごとに切り替え可能
   - 太郎本体を触らず実験条件を変えられる

B0〜B3は達成のための**工学的な刻み**であって、人間の発達段階名ではない。

| 段階 | 名前 | 内容 | 成功基準 |
|---|---|---|---|

| **B1** | 言葉が状況に伴う（受信） | 親が食事時に「まんま」。太郎は言葉＋状態変化を同時体験 | 「まんま」後に状態改善を予期できる |

## 8. モデルの保存と学習方針

### 確定方針

1. **モデルA（内部状態なし版）を学習済みで保存** → 比較用ベースライン（保存済み）
2. **モデルB（身体シミュレーションあり版）はゼロから連続学習**
   - 泣き→クーイング→喃語→オウム返し→初語と途中リセットなし
   - 人間模倣として正しい（赤ちゃんの発達は途切れない1本の弧）
   - 島皮質の追加で脳の形が変わるため、A保存→ロードは不自然
   - Bの学習中にオウム返しが保たれることを毎回確認できる（切り分け）

### 破壊的忘却との関係

わざとプログラムするものではない。太郎の脳は1セットの重みを共有するGRUなので、
Bの学習がAの記憶を上書きする可能性がある（自然に起きてしまう現象）。
起きたら海馬（睡眠リプレイ）の実装動機になる。

## 9. 予想される壁

### 壁1：破壊的忘却

初語を学ぶうちにオウム返しが劣化する可能性。
対策：監視→海馬（hippocampus.py）実装。CLS（McClelland, 1995）。

### 壁2：意味の検証が難しい

対策：§3の4条件で操作的に検証。

### 壁3：これは「真の意味」か、ただの条件反射か

初期の人間の語学習も大部分は連合（条件づけに近い）。
連合から始めるのは人間模倣として正しい。
より高度な意味（志向性・離脱表現）はさらに後の段階。

### 壁4：報酬のバランス

homeostasis・模倣・予測・社会的報酬の重みづけ。
「困ったらパラメータ調整」ではなく、
まず「人間のこの時期、どの動機が優勢か」を調べる。
