# 目標D 設計：他者理解（触覚で相手を知覚、視覚なし）

作成：2026-07-14
位置づけ：C で確立した自己モデル（順モデル＋C3の逆モデル/Goal Babbling）を、**触覚で知覚した他者に当てはめて理解する**段階。C のソースコードを土台に D/scripts で書く（D は自前の脳を持たず C を流用）。

## ゴール
自分の内部モデルを他者に転用し、**相手の行動を認識し、次を先読みする**。視覚なし＝独自方針（先天盲の他者理解＋介護の hands-on に対応）。最終的に介護AI（触れて支え、相手の動きを先読みする）へ。

## 現状（`D/README.md`・既存scripts より）
| 項目 | 状態 |
|---|---|
| 環境 `two_agent_env`（2体MIMo、A=太郎が**触覚でB=相手を知覚**） | ✅ 実装済み |
| 認識（どの行動か、2AFC） | ✅ 84%／触覚100%（易しい課題） |
| **先読み（相手の次の行動を予測, `d_b2`）** | ❌ **~55%≒偶然（核心の未達）** |

## 診断：なぜ先読みが失敗するか
- **相手の次の行動は"相手の意図/方策"が生む**もの。太郎の状態z（物理の結果）だけからは読めない。
- 旧 `d_b2` は **相手の意図を推論してない**（状態z→次行動を線形読み出しだけ）＋**相手(m2)がランダムな運動探索**（そもそも読むべき"意図"が無い）。
- ＝先読みには「**相手を"自分のように"モデル化して、その意図/目標を逆算する**」（MOSAIC/能動的推論）が要る。＝**C3で得た逆モデルが、まさにこの武器**。

## 触覚の制約（設計を形作る）
- 触覚は**接触時のみ**。非接触＝相手を知覚できない（視覚と違い遠隔不可）。
- ＝他者理解は「**触れている間**」に限られる。
- **人間的**：先天盲は触覚＋聴覚で他者理解（触覚は接触が要る＝自然）。**介護は hands-on**（触れて支える）＝この制約はむしろ介護に忠実。

## C→D アプローチ（C3の逆モデル/Goal Babbling を使う）
```
1. 相手(B)を Goal Babbling で動かす（目標を持って動く＝読むべき"意図"を生む）
2. 太郎(A)が触覚でBを知覚（two_agent_env の touch_of_B）
3. 逆モデルで「相手はどの目標を狙ってる？」を逆算（MOSAIC＝どの目標がこの行動列を生むか）
4. その目標に向かう「相手の次の行動」を予測 → anticipation が偶然を超えるか
```
＝旧d_b2（順モデルの状態だけ）が失敗した先読みを、**C3の逆モデルで相手の意図を推論**して攻める。＝「Cの自己モデルを他者に当てはめる」の本命。

## 指標（既存 `d_b2` の対照を流用）
- **anticipation 2AFC** が **persist（滑らかさ）・floor（未学習）・shuffle（ラベル混ぜ）・50%** を明確に超えるか。
- 認識（どの行動か）も併記。※自己他者・転用は**復元でなく区別/認識が頑健**（C3の教訓）。

## 根拠（ラベル）
- **【人間模倣】** MOSAIC（Haruno/Wolpert/Kawato＝順逆ペアで他者の行動認識・模倣）／能動的推論の action understanding（相手の行動を自分の生成モデルの推論で理解）／先天盲の触覚・聴覚ベースの他者理解／介護の hands-on。
- **土台＝C**：G1（自己予測）・G2（自己/他者の区別）・C3（逆モデル/Goal Babbling）を他者へ転用。

## 段階（改訂 2026-07-15：D0を新設。二体は「太郎アルファ（学ぶ側）／太郎ベータ（相手）」と呼ぶ）

**改訂の理由**：①Cの自己モデル(+50)は**触覚を一度も感じたことがない**（内受容+固有感覚+前庭のみ）＝触覚を知らないモデルに他者の触覚を渡しても解釈できない＝共鳴マップが成立しない。②実測で**自由なMIMoは倒れて互いにほぼ触れない**（接触率2%、触覚の97%は床）＝「たまたま接触」は原理的に不成立。③分野標準（MIMo `selfbody` / BabyBench 2025）は**自己接触**が課題で、**座位＋weld固定＋腕のみ自由**で倒壊も床混同も解いている。＝**自己接触で触覚を接地してから他者へ**が発達順序としても実装としても正しい。

- **D0（新設）自己接触**：MIMo公式 `MIMoSelfBody-v0`（座位・`<weld body1="mimo_location"/>`で固定＝倒れない・触覚ON・**右腕のみ自由**）で、太郎の学習（運動性喃語＋予測＋睡眠リプレイ）により**触覚を含む自己モデル**を確立。実寸＝固有感覚226/触覚1908/内受容4/行動8。
  - 予測対象に**触覚を含める**＝「こう動かせば自分の体にこう触れる」＝**触覚的な身体スキーマ**。これが後の「予期しない触覚＝自分のせいでない＝他者」（agencyの触覚版）の土台になる。
  - 床は尻・脚で感じ、**手は床から浮く**＝どの部位で感じたかで発信元が分かる（somatotopy）。触覚は`infant_touch_params`で乳児相当の粗さに（比率＝手/指が最密は保つ）。
- **D1：2体**（`d_env.TwoMimoEnv`＝`MIMoV2DummyEnv`継承、接頭辞`beta_`でベータをattach）。ベータをアルファの**腕の届く位置に座位で配置** → アルファの手がベータに触れる（介護のhands-on）。
- **D2**：逆モデルで相手の目標を推論（触覚観測から）。α=共鳴/β=観測モデルを併走。
- **D3**：先読み（anticipation）が偶然/persist/floor/shuffle を超えるか＝Dの核心。
- 各段は太郎の標準ワークフロー（事前登録→比較→アブレーション）で。

**視覚・体温（保留中の候補）**：MIMo公式の自己接触も**視覚なし**（`vision_params=None`）＝この段階は視覚不要が分野標準。他者段階で触覚が足りなければ、①**視覚**（`d_env`は`vision_params`を渡すだけで有効。ベータは実体なので普通に映る）②**体温**（人の皮膚は温度受容も持ち、ロボット皮膚でも熱センシングで"人との接触"を同定するのが確立手法）を足す。**触覚のみで試し→ダメなら追加**の順で判断する（実験結果を根拠に決める。要検証の論点として別途整理済み）。

## 未解決・注意
- 相手の行動がランダムだと目標推論が空回り → **相手を Goal Babbling で"意図的"に**するのが前提。
- 触覚は接触時のみ → 先読みは接触が続く場面に限る（介護的には自然）。
- C の逆モデルは"区別"は頑健だが"絶対復元"は弱い（§5）→ 先読みも**区別/確率**で測る（絶対予測でなく）。
