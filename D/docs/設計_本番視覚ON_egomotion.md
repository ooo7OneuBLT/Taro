# 設計：本番環境で視覚をON（太郎の本物の脳でegomotion割引を創発させる）

作成：2026-07-17／位置づけ：D（他者理解）。門番（教師あり・使い捨て）で「解ける問題」と確認できた
egomotion割引を、**太郎の本物の脳（taro_core・本能だけ・正解ラベルなし）**に載せる段階。

## これまでの経緯（なぜここに来たか）
1. 触覚だけの他者知覚＝**構造的限界**（相手が触覚の3.9%・床82.5%に埋もれ、容量を増やしても頭打ち＝情報不足）→視覚へ。
2. 視覚は太郎が静止なら距離/左右/速度が読めるが、**太郎が動くと崩壊**＝自己運動(egomotion)と対象運動が画像だけでは分離不能（この期間の最重要発見）。
3. 人間は前庭覚/固有感覚/遠心性コピーで自己運動を割り引く→**合成版で ④画像+体の感覚=100% > ②画像だけ85%** とクリーンに確認（門番＝教師あり使い捨てNN。「問題が公正か」の確認用で、太郎の外に置いた）。
4. → 本番＝この割引を**太郎の予測本能で（教師ラベルなしで）自力獲得**できるか。

## 中核の設計思想：専用機構を作らず、"既にある予測本能"に視覚を見せるだけ
太郎の脳はもともと「行動→次の固有感覚」を予測する順モデル。ここに視覚を足し、**「行動→次に見える映像」も予測させる**と——
- 自分の動きで説明できる映像のズレ＝**予測できる**＝自己運動
- 予測から外れたズレ＝**相手の本当の動き**

＝**egomotion割引は"予測の残差"として自然に立ち上がる**。von Holstの**再求心性原理(reafference＝遠心性コピーで自己由来の感覚変化を打ち消す)**そのもの。**【人間模倣】**。しかも先行研究の主張（Nagai 2019「予測学習が発達の駆動因」）と同じ土俵。

## 学習原理の線引き（教師あり vs 自己教師あり）
禁じるのは「ターゲット全般」でなく「**創発させたい答えそのものを外から注入するオラクル**」。判定は**「本物の赤ちゃんがその信号を持っているか」**：
| 信号 | 赤ちゃんは持つ？ | 太郎で可？ |
|---|---|---|
| 予測誤差（自分の行動→次に起きた感覚） | ✅ | ✅ 自己教師あり |
| 模倣（観察した対象に合わせる） | ✅ | ✅ |
| 報酬（快・不快） | ✅ | ✅ |
| 「これは自己運動/これは他者」ラベル | ❌ | ❌ ＝門番が使った信号。だから太郎の外に置いた |
egomotionは「行動→次の映像」を予測する自己教師ありで足りる＝ラベル注入は不要（むしろ予測学習の枠組みから外れる）。

## 段階プラン（1機構ずつ・検証してから次）
- **Stage 1**：視覚ON＋単純連結で融合に入れる＋視覚を予測対象に。自己モデルが壊れず学べるか＋視覚が情報を足すか（ベースライン）。**窓なし**（GRUの自然な再帰記憶だけ）。
- **Stage 2**：融合をcross-attention（体の感覚→視覚）へ。egomotion割引が予測誤差だけで創発するか（門番の結果を本能で再現）。
- **Stage 3**：Dの他者理解・先読み(anticipation)へ接続。

## ★事前登録：Stage 1が天井に当たった時の「第一の一手」＝窓（複数コマ誤差回帰）
Stage 1が頭打ちしたら、**真っ先に「窓（PV-RNNの複数コマ誤差回帰）不足」を疑う**（＝事前登録した第一仮説）。根拠：
- 合成版で「1コマで足りた」のは**②雑音ゼロの体感覚信号＋③常に中心スタート**という2つの理想化のおかげ（後者は物理版で見つけた「出発位置がラベルを漏らす」交絡の裏返し）。本物の環境（既知の出発点なし・信号にノイズ）では成り立たない。
- 窓の**機能**（時間統合＋過去の解釈をあとから書き換える＝postdiction。フラッシュラグ錯覚等）は**より人間的**。ただし**実装**（K枚バッファ＋潜在の再最適化）はCNN/GRU同様の工学的近似(⚠️)。
- 実装は**中くらい**：既存`taro_core/src/brain/hippocampus.py`のFIFOバッファ（貯める→まとめて使う→捨てる）のパターンを流用できる。ゼロからの発明ではない。
- ※今は作らない（必要と分かってから）。GRUは元々隠れ状態で時間統合しているので、Stage1が窓なしで通る可能性もある。

## 実測の裏付け（「時間がかかりすぎる/教師ありが要る」懸念の否定）
- **速度**（`D/scripts/bench_vision_speed.py`実測）：視覚OFF 349サンプル/秒、**視覚ON(64×64) 243サンプル/秒＝1.4倍遅いだけ**。門番が天井に達したn=400を集めるのに視覚ONでも**約1.6秒**。＝「絵本を何ヶ月も読む」レベルの遅さではない。低視力(低解像度)の人間模倣が、そのまま計算削減にもなっている。
- **学習量**：C段階で同種の頭打ち（オンライン1回きり学習）を**睡眠リプレイ**（同経験を平均43回復習・自己教師ありのまま）で突破済み。`hippocampus.py`のバッファは汎用で、視覚込み経験も**無改造で**貯めて復習できる。＝量の問題は既存インフラの延長で対応可能。
- 留保：上記はサンプル"生成"速度で、学習(CNNの逆伝播)コストは別。ただし物理的ボトルネック(レンダリング)が軽いと分かったのが収穫。門番のn=400で太郎の本物の脳が足りる保証はない。

## 実装タスク（この設計での順序）
0. ✅**完了(2026-07-17)**：MinimalFusion を `C/scripts/run_c_metrics_ac_lr.py` から
   `taro_core/src/senses/fusion.py` へ抽出。視覚オプション`vision_res`を追加（触覚`touch_dim`と
   同じ後方互換の作り）。run_c_metricsは`from fusion import MinimalFusion`で再エクスポート＝D側8本は無改修。
   **発見**：視覚込み融合は既に`tests/test_phase8_motor_learning.py`の`SensoryFusion`にあり（5感覚→連結→
   layer_norm）、そのロジックを踏襲＝作り直さず統合。検証：視覚なし192次元(=従来C同一)／視覚あり256次元。
1. 【次】`TwoMimoEnv`（フルボディ相手）に視覚ON＋MIMo純正acuityで、太郎の眼球視界を取得。
   ※現C学習ループ(run_c_metrics)は単体env(TaroSupine/Lean)前提。2体env＋視覚を学習ループに繋ぐ
   統合方法は要検討（ここが最大の実装判断・リスク箇所。着手前に相談）。
2. `MinimalFusion(vision_res=…)`を学習ループで使う（フラグで視覚ON。単純連結）。解像度は低視力(例64、要感度確認)。
3. 予測対象に視覚（または視覚の位置/変化）を追加＝reafference。
4. 学習ループで回し、除去テスト（視覚OFF・acuity有無）＋アブレーションで検証。

## 未解決・要検証（⚠️定数と除去テスト）
- 解像度(64等)・acuity年齢は⚠️定数→感度確認。低くしすぎると相手が見えない懸念。
- 「視覚を予測対象に」の具体形（生画像を予測 vs 位置/オプティックフローを予測）は要検討。生画像19万次元をそのまま扱うと学習が不安定（合成版で確認済み）→位置/変化に絞る案が有力。
- 予測対象に視覚を足すと、固有感覚予測（従来のG1）とのバランスをどう取るか。
