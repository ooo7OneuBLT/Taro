## 2026-07-16

### egomotion割引の門番②：左右移動版を作る過程で3つの設計バグを発見・修正

C日誌の「egomotion割引の仮説を門番で検証」の続き。今回は**左右移動**（親=左右／太郎=頭yaw）
で実装（`d_ego_leftright.py`）。近づく/遠ざかるでなく左右にしたのは、軸を揃える（親も太郎も
左右）ことで egomotion（自己運動の視覚的ズレ）と対象運動（相手の真の動き）が**同じ見え方を
偽装する**、最も厳密な設計にするため。

**バグ①：腕を動かす自己運動は空振り**（ユーザー指摘で発覚）。太郎は仰向けで目は真上、
腕は視界に入らない＝腕を動かしても眼視界はほぼ変化しない（実測 eyechg 28→29、ほぼ同じ）。
頭yaw（首を振る）に変更したところ、眼視界変化 18.5〜36.6 と大きく動くことを確認。

**バグ②：フィルタが生存バイアスを作る**。「相手が全コマ写るクリップだけ採用」というフィルタが、
親の左右移動と頭yawが**同じ軸**を共有するせいで、「頭の向き」と「親の真の向き」を相関させて
しまう（ユーザー指摘「そもそも視界に親がいない状況が存在するんだ」で根本原因を特定）。
結果：体の感覚だけで正解率100%（本来~50%であるべき）というカンニングが発生。

**バグ③：視野角60度が原因で頭yawが即座に視界外を作る**（実測：頭yawだけで57%のコマが
視界外）。太郎の目カメラは元々fovy=60固定で、`vision_params`の`fovy`はacuity計算にしか
使われず**実際の描画に反映されていない**バグも同時に発覚（`model.cam_fovy[cid]`を明示的に
設定するよう修正）。視野角を120度まで広げたところ、頭yaw0.40での視界外率が57%→0%に改善。
ただし親のカプセルサイズが大きすぎる(0.14+)と太郎自身の腕と物理衝突する事故も発見（腕の
contype/conaffinityは有効なまま）→サイズ0.10で安定と確認。

**バグ④（最も重要）：赤判定の閾値が絶対値で厳しすぎた**（ユーザー指摘「赤いのにOUT OF VIEW
って表示、おかしくない？」で発覚）。旧判定 `R>100 & G<80 & B<80` は、画面端で暗くなった
養育者（実測RGB=100,32,32）を「赤でない」と誤判定していた。背景は青系(R<G<B)なので、
絶対値でなく**相対的な赤み**（`R - (G+B)/2 > 25`）に変更。修正後、fovy120・size0.10の設定で
親だけ左右／頭yawだけ／両方同時（最悪ケース）のいずれも**視界外0%**を確認＝生存バイアスは
原理的に解消。

**教訓**：4つのバグは全て**ユーザーが動画・数値を実際に見て「おかしい」と気づいた**ことが
発見の起点だった（[[feedback-watch-dont-just-measure]]の一貫）。特にバグ④は、私が「視界外
49%」という数字をそのまま解釈しかけていたところを、ユーザーの一言が救った。

**最終設定**：解像度64・fovy120・養育者サイズ0.10・左右移動幅±0.06・頭yaw振幅0.25〜0.45・
赤判定は相対赤み方式。この設定で本番測定（n=200、5条件：①静止画像のみ②自己運動画像のみ
③自己運動・体の感覚のみ④自己運動・画像+体の感覚⑤④のシャッフル）を実行中。結果は追記予定。

**本番測定（n=200）の結果**：
| # | 条件 | 正解率 |
|---|---|---|
| ①天井 | 静止・画像だけ | 100.0% |
| ②床 | 自己運動・画像だけ | **100.0%（下がらず）** |
| ③検査 | 自己運動・体の感覚だけ | **50.0%（想定通り）** |
| ④本命 | 自己運動・画像＋体感覚 | 100.0% |
| ⑤ | ④のシャッフル | 100.0% |

**③=50.0%＝生存バイアス完全解消（4バグ修正の成果）。だが②が下がっていない＝そもそも
今回の頭yawは左右信号を隠しきれていない**。前回の「近づく/遠ざかる」（自己運動で98%→54%に
崩壊）と違い、頭yaw単体では判定器にとってまだ易しすぎた可能性。④=100%は「体の感覚が
効いた」のでなく「②に伸びしろが無かった」ため、本命の問いにまだ答えが出ていない。

### 続き：頭yaw強化の失敗→ユーザーの「相対速度」洞察で突破（2026-07-16深夜〜17早朝）

**頭yaw強化(0.6-1.4)は失敗**：②が100%のまま下がらず、③が63%に悪化（カンニング再発）、
採用率50%（生存バイアス再発）。動画で確認すると、強く振ると①自分の腕が視界に写り込む
②親の移動が頭yawに埋もれ両ラベルが似る。ここで「反転を起こすほど強く振る↔視界外/腕写込み/
生存バイアス」というジレンマにぶつかった。

**★ユーザーの突破口**：「頭yawに上限をかけて、親をそれより小さく動かせばいい」。反転は
"頭yawの見かけ移動 vs 親の実移動"の**相対速度**の問題なので、親を小さくすれば弱い頭yawでも
反転する——私が「1変数(頭yaw)だけを押し続け、親の移動幅を固定と思い込んでいた」盲点を突かれた
（→ [[feedback-reframe-discipline]] として記録）。実測：頭yaw0.8固定なら親幅±0.06〜0.01の
どれでも反転（自己運動時に見かけ移動が逆符号）＆視界外0＆静止では読める、の3条件が両立した。

**加えて位置漏れも発見・対処**：右=常に左端スタート/左=常に右端スタート、で「出発位置」が
ラベルを教えていた。→ 出発位置をランダムにずらし「動きの向きだけがラベル」になるよう修正。

**最終設定と結果（親幅0.04・出発位置ランダム化±0.03・頭yaw0.7-0.9、n=80）**：
| 条件 | 前回 | 今回 |
|---|---|---|
| ①静止・画像だけ | 100% | 100% |
| ②自己運動・画像だけ | 100% | **93.8%（初めて下がった）** |
| ③体の感覚だけ | 63% | **56.2%（ほぼ50%＝カンニングほぼ解消）** |
| 採用率(自己運動) | 50% | 58% |

**残る課題＝視界外がまだ4割**：却下クリップの集計で「親と頭yawが同じ向き」のとき視界外に
なりやすいと判明（30試行中の却下10/11がこのパターン）。＝生存バイアスの源がまだ残り、③が
50%より僅かに高い(56%)残り火の正体。**視界外を0にすれば③は完全に50%・②ももっと下がるはず**。

### 転換：物理エンジンをやめて「合成データ」で直接テストする方針へ（2026-07-17 早朝）

**ユーザーの三人称視点での指摘で、私の誤説明が発覚**：`lr_selfmove_right.mp4`（ラベルright）
で「三人称で見ても親が左に動いている」との指摘。実測すると right=True で三人称centroid_xが
155→148（画面左へ）。私は直前に「これはegomotionによる見かけの反転（見た目だけ左・真実は右）」
と説明していたが**完全な誤り**だった。真相は「三人称カメラの向きが太郎の眼視界と左右逆なだけ」
（眼視界では right=True で右に動く＝29→41で確認済み）。＝ラベル自体は太郎の眼視界基準で正しい
が、三人称動画では逆に見えて紛らわしい。**私が「反転が起きたはず」という思い込みで動画を都合
よく解釈した**＝[[feedback-reframe-discipline]]の教訓の再発。ユーザーの直接観察が正した。

**ユーザー提案＝合成データで直接テスト**：「検証なのだから、赤丸が動く映像＋自己運動信号を
直接合成して送りつけ、判定器が真の動きを復元できるか見ればいい」。物理エンジンで反転を起こす
と、視野外・自分の腕の写り込み・生存バイアスが絡み合い続けた。合成なら egomotion割引の本質
だけを取り出せる。→ `d_ego_synthetic.py` を作成：見かけ位置 = 中心 + 真の動き − 自己運動、
真の動きと自己運動を独立に抽選、画像4コマ＋自己運動信号を判定器へ。

**合成版・初回結果（n=400）＝判定器にバグ**：
| 条件 | 正解率 |
|---|---|
| ①静止相当・画像だけ(天井) | **52.5%（バグ・天井が出ていない）** |
| ②自己運動・画像だけ(床) | 78.8% |
| ③自己運動信号だけ | 51.2%（○ 独立性OK） |
| ④画像＋自己運動信号(本命) | 97.5%（一見good） |
| ⑤④のシャッフル | 98.8%（下がるべきが下がらない） |

①が天井にならない原因＝**判定器に生画像49152次元をそのまま渡して過学習/学習不安定**。本物の
実験では赤い塊が大きく位置相関が強くて偶然読めていたが、合成の小さい赤丸(radius5・移動6-14px)
では埋もれた。**④=97.5%は一見「体の感覚で割り引けた」ように見えるが、①天井が52.5%で壊れて
いる以上、額面通りには受け取れない**（判定器が不安定なので、②③④⑤の相対比較の土台がない）。
⑤シャッフル98.8%（順序不要）も気になる：見かけ位置apparent[t]＋自己運動信号body[t]＝真の
位置、が各コマ単独で成り立つため順序が要らない、という設計上の性質と思われる（これ自体は
egomotion割引の本質＝見かけ−自己運動＝真、と整合的だが、①が壊れている今は確証にならない）。

**合成版の残る修正点**（後日の宿題）：
- 判定器の入力を「生画像」→「赤丸の位置系列（少数次元）」にする＝過学習を避け天井を出す。
  画像は可視化用にだけ描く。
- 可視化が弱い（赤丸が小さく動きが小さい）＝大きく・動きを大きくして映像で分かるように。

**今日一日の総括**：egomotion（自己運動と対象運動の分離）を「使い捨て判定器」で測る設計を、
物理版で徹底的に詰め（4バグ発見・修正、ユーザーの相対速度洞察で②を初めて崩す・③をほぼ50%へ）、
最終的にユーザー提案で「物理を排した合成版」という、より本質的なアプローチに到達した。合成版の
判定器バグと可視化を直せば、egomotion割引（体の感覚で自己運動を割り引き真の動きを読めるか）を
クリーンに測れる見込み。

### 合成版バグ修正2つ→ egomotion割引を初めてクリーンに支持する結果（2026-07-17 早朝）

**修正①：動きを大きく**（ユーザー指摘「動きが小さすぎて見えない」）。RES 64→128、真の動き幅
6-14→15-28、自己運動幅0-20→0-35、丸半径5→9。これだけで①天井が52.5%→**100%**に回復
（動きが小さいと画像から向きを読む学習自体が不安定だった）。

**修正②：判定器に生画像でなく「位置の系列」を渡す**。動き拡大後も④(画像+信号)=52.5%が
②(画像だけ)=81.2%より低く、しかも⑤シャッフル=88.8%が④より高いという逆転が発生＝
**生画像19万6608次元にわずか4次元の自己運動信号を混ぜると学習が不安定になる**証拠（19日の
「スケールが大きい特徴に学習が引きずられる」問題の親戚）。→ `extract_x()`で赤丸のx座標を
先に抽出（物理版の`_red_pixels`重心と同じ発想）、判定器には「位置の系列＋自己運動信号」という
小さな問題として渡すよう変更。

**最終結果（n=400）**：
| 条件 | 正解率 |
|---|---|
| ①静止相当・画像だけ(天井) | 100.0% |
| ②自己運動・画像だけ(床) | 85.0% |
| ③自己運動信号だけ(検査) | 53.8%（○独立性OK） |
| ④画像＋自己運動信号(本命) | **100.0%（②より+15pt）** |
| ⑤④のシャッフル | 100.0%（下がらないが設計上妥当※） |

※各コマの見かけ位置と自己運動信号の組がその場で「真の位置＝見かけ−中心＋信号」を一意に
決める設計のため、時間順序は本来不要（速さ実験で見た「間隔だけで分かる」情報と同じ構造）。
下がらないこと自体は矛盾でない。

**★意味**：④が②より明確に上がった＝**「体の感覚（自己運動信号）を足せば、egomotionで
崩れた画像だけの読みを立て直せる」という仮説を、初めてクリーンな形で支持するデータが出た**。
物理版で4つのバグと格闘した末、合成版でようやく本質を検証できた。ただし合成は理想化された
設定（見かけ位置＝真の動き−自己運動、という関係を厳密に仮定）なので、物理版（実際のMuJoCo
シミュレーション）でも同じ効果が出るかは別途確認が要る。

**次にやるべきこと（更新）**：
1. 【最優先確認】この合成版の結果を物理版でも再現できるか。物理版は視界外・生存バイアスが
   残っている（採用率58%、③が56%で50%よりやや高い）ので、まず物理版側の視界外を0にする
   （出発位置ばらつき±0.03→±0.015、頭yaw上限微減、親をさらに小さく、のいずれか）。
2. 物理版で④>②かつ③≈50%かつ⑤<④が確認できれば、egomotion割引の仮説が実機シミュレーションで
   確定。Phase5視覚エンコーダ＋多感覚融合を太郎の本物の脳へ接続する設計に進む。
3. 静止画版（近づく/遠ざかる）でも同じ交絡（視野角・赤判定・出発位置・生存バイアス・生画像vs
   位置抽出）の洗い直しをする価値がある。

## 2026-07-18〜19：Dの詰まりの正体＝「発達順の飛ばし」と判明 → 運動発達を目標Eに分離

### 経緯：egomotion実装の前に、そもそも太郎の運動が非人間的と発覚（C5へ寄り道）
合成版でegomotion割引を支持するデータ（④画像+体感覚＝100%）は出たが、これは**使い捨ての門番（教師ありNN）での確認**で、太郎の本能の脳には未実装。実装に進む前にDの追加学習で太郎の眼球視界を動画で見たところ、**視野が激しく揺れる＝運動生成が非人間的（カクつき過大）**と判明。→ **C5（運動品質の調査）へ寄り道**し、progress報酬（学習進度＝Oudeyer）＋新生児体（成長モジュール）で mean|jerk| 4690→1636（65%減）を達成（C側の成果、`c5_progress_seed0.pt`）。

### 本物との比較で核心が見えた（動画目視）
新生児の自発運動（General Movements＝writhing）の本物動画（[Normal General Movements](https://www.youtube.com/watch?v=XrVZmG9KSxo)）と太郎を見比べた：
- **本物**：ほぼ足が少し動く程度・頭はほぼ動かない・低〜中振幅でなめらか
- **太郎**：全身が大きくぴくぴく・頭まで動く

→ 太郎の運動は振幅・速度が過大＝GMAでは"異常寄り"のサイン。そして**頭が動く→眼球視界が揺れる**のが egomotion 詰まりの一因。

### 依存関係を文献で裏どり → Dは「時期尚早」だった
[発達依存関係マップ](../../doc/発達依存関係マップ.md)を作成（Hadders-Algra 一次→二次変動性／Meltzoff "like me"／遠心性コピー・前庭／Oller）。結論：
- **他者理解(D)は「運動性喃語の終了（目的的運動への移行）＋自己運動の割り引き」が前提**。
- 太郎はそれらを飛ばして「動く自分での視覚的他者理解」をやった＝**依存を飛ばした"時期尚早"**（「最初から無理」ではない）。
- ＝Dがegomotionで詰まったのは、間違った発達順の帰結。

### 決定：運動発達を目標Eに分離、Dは"待ち"
運動発達（運動性喃語→目的的運動→…→歩行）はC・Dの土台なので、独立目標**E**に切り出した（[E/README_E.md](../../E/README_E.md)・[E研究日誌](../../E/docs/研究日誌.md)）。**Dは温存し、E（運動性喃語の終了＋egomotionの脳内実装）が済むまで"待ち"**。次はEの全体設計＝「運動性喃語がどう終わるか（一次→二次変動性の移行）」から。
